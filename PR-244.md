# PR-244: Public timetable view generation

## Summary

This PR performs roadmap PR-6: generate one public timetable view from canonical timetable data plus publication policy.

## Added

- `src/data/publicationDisplayPolicies.json`
- `scripts/timetable/build-public-timetable-view.mjs`
- `data/generated/timetable/public/meeting-list.json`
- `data/generated/timetable/public/meeting-details.json`
- `src/lib/timetable/publicTimetableViewModel.ts`
- `scripts/check-public-timetable-view.mjs`

## Changed

- `src/data/publicationDisplayPolicies.ts` now reads the shared JSON policy source.
- `scripts/check-publication-policy-resolver.mjs` now validates that shared policy JSON.

## Result

The public builder reads canonical meetings and canonical meeting details, applies the publication policy, and writes one meeting-list dataset and one meeting-details dataset.

List output remains one meeting per row and never contains race-by-race rows.

C rows hide first and last race times.

B rows may show first race time only.

B+ rows may show first and last race times.

A detail rows show race label and post time only.

A+ programme-summary fields are retained only when effective public rank is A+.

The initial HKJC A+ detail keeps race name, distance, surface, and course where present.

The current JRA A detail keeps only race label and post time.

## Scope

No page input is migrated in this PR.

No list page is changed.

No country or racecourse page is changed.

No meeting detail page is changed.

No source fetching is added.

No legacy input is deleted.
