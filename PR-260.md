# PR-260: Public timetable list API and scoped component contracts

## Summary

This PR performs roadmap PR-9: prepare reusable public timetable list filters and scoped component contracts before country or racecourse page insertion.

## Added

- `src/lib/timetable/publicTimetableFilters.ts`
- `src/components/PublicTimetableScopedList.astro`
- `docs/specs/public-timetable-list-contract.md`
- `scripts/check-public-timetable-list-contract.mjs`

## Result

Reusable public timetable filter helpers now exist for:

- date
- country
- racecourse
- date range
- generic scope
- date grouping
- limit / sorting

A scoped list component now exists for later country and racecourse timetable sections.

The display contract keeps one meeting per card and does not expose race-level rows or A+ programme-summary fields on list surfaces.

## Scope

No country page is changed.

No racecourse page is changed.

No meeting detail page is changed.

No source fetching or snapshot acquisition is added.

No legacy input is deleted.

Country and racecourse insertion can remain delayed until those page specifications are stable.
