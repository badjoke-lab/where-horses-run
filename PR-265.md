# PR-265: Meeting detail page public view migration

## Summary

This PR performs roadmap PR-10: move the meeting detail page from transitional normalized detail input to public meeting-details output.

## Changed

- `/timetable/meetings/{meeting_id}/` now reads public detail records through `publicTimetableViewModel.ts`.
- Static paths are generated only for meetings that have public detail records.
- The page displays effective public rank, capability rank, policy id, source status, first race time, last race time, and official source.
- A detail rows show race label and post time only.
- A+ detail rows may show policy-approved race name, distance, surface, and course fields.
- Add `scripts/check-meeting-detail-public-view.mjs`.

## Result

Meeting detail pages now read public meeting-details.json through publicTimetableViewModel.

The old transitional detail source is no longer used by the page.

Publication boundary language explicitly excludes entries, runners, horses, jockeys, trainers, odds, results, payouts, predictions, tips, and raw source pages.

## Scope

No country page is changed.

No racecourse page is changed.

No source fetching or snapshot acquisition is added.

No legacy input file is deleted.
