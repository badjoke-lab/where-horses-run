# PR-255: Core timetable list stabilization

## Summary

This PR is the revised roadmap PR-8.

It stabilizes the public-view based display used by calendar, today, tomorrow, and current timetable pages before any country or racecourse page insertion.

## Changed

- Upgrade `TimetableMeetingList.astro` from a compact inline row into a mobile-safe meeting card list.
- Keep one card per meeting.
- Keep first/last race time display controlled by effective public rank.
- Keep detail links nullable and only shown when public view provides `detail_path`.
- Keep official source links visible on every meeting card.
- Add source status and last checked metadata when available.
- Add `scripts/check-core-list-stabilization.mjs`.

## Result

Core calendar, today, tomorrow, and current timetable pages keep using public view rows.

The shared list component now gives a stable visual contract that can later be reused by country and racecourse timetable sections.

A/A+ list pages still do not display race-level rows, race names, distance, surface, or course fields.

Country and racecourse pages are intentionally not changed in this PR.

## Scope

- No country page change.
- No racecourse page change.
- No meeting detail page change.
- No source fetching or snapshot acquisition.
- No legacy input deletion.

## Next

Next roadmap item is PR-9 shared list API and scoped component contracts.
