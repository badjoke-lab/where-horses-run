# PR-266: Meeting detail page public view migration

## Summary

This PR performs roadmap PR-10: migrate meeting detail pages from transitional normalized detail inputs to the public meeting detail view.

## Changed

- `/timetable/meetings/{meeting_id}/` now reads from `publicTimetableViewModel.ts`.
- Static paths are generated from public timetable rows that expose `detail_path`.
- Meeting detail data is read from `data/generated/timetable/public/meeting-details.json` through `getPublicTimetableMeetingDetail()`.
- `normalizedTimetableMeetingDetails.ts` direct reading is removed from the page.
- `normalizedTimetableCalendarPreview.ts` detail-path helper is removed from the page.
- `scripts/check-meeting-detail-public-view.mjs` validates the public detail boundary.

## Result

Meeting detail pages now read from public meeting-details.

A detail rows show race label and post time only.

A+ detail rows show only policy-approved programme summary fields.

The current JRA detail fixture stays A and does not expose race name, distance, surface, or course fields.

The current HKJC detail fixture stays A+ and may expose race name, distance, surface, and course according to policy.

Official source and last checked metadata come from public detail output.

## Scope

No country page is changed.

No racecourse page is changed.

No source fetching or snapshot acquisition is added.

No legacy input file is deleted.

No racecard, entry, runner, horse, jockey, trainer, odds, result, payout, prediction, tip, raw source HTML, stream URL, or replay mirror data is displayed.
