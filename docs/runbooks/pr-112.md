# PR-112 — South Africa / South Korea parser-backed promotion

## Purpose

PR-112 continues the 13-country D-to-C/B promotion work after PR-111.

The goal is to move South Africa and South Korea away from D-only source target status by using fixture-backed parsers that produce normalized promotion records.

## Scope

Included:

- South Africa race-day fixture rows for Turffontein, Greyville, Fairview, and Scottsville.
- South Korea KRA race-day fixture rows for Busan-Gyeongnam, Seoul, and Jeju.
- Shared simple race-day row parser.
- Generator that emits normalized South Africa / South Korea promotion records.
- Validator that proves both countries produce Level B records.
- Package check integration.

## Files added

- `data/fixtures/timetable/south-africa-race-day.html`
- `data/fixtures/timetable/south-korea-kra-race-day.html`
- `scripts/timetable/parsers/simple-race-day-rows.mjs`
- `scripts/generate-pr-112-sa-korea-promotions.mjs`
- `scripts/check-pr-112-sa-korea-promotion.mjs`

## Package script added

- `validate:pr-112-sa-korea-promotion`

Included in `npm run check`.

## Data level handling

This PR does not claim Level A.

The fixture rows expose meeting date, racecourse, and first race/post time. That supports Level B promotion. Full race rows require source-specific racecard parsers before Level A can be claimed.

## Source handling

South Africa has multiple relevant source families. This PR treats operator rows as promotion rows and keeps NHRA as a regulator/supporting source rather than the direct timetable source.

South Korea uses KRA as the official source family for Seoul, Busan-Gyeongnam, and Jeju rows.

## User-facing meaning

South Africa and South Korea should no longer be treated as D-only in the 13-country major racing scope.

The promoted rows can support display of:

- country
- system/group
- racecourse
- meeting date
- first race/post time
- source trace
- source capture date

## Operational meaning

PR-112 repeats the parser-backed promotion pattern used in PR-110 and PR-111:

1. Store source-family fixture rows.
2. Parse meeting date, racecourse, and first time.
3. Generate normalized promotion records.
4. Validate that the records are Level B and do not claim Level A.

## Non-goals

This PR does not add live fetch, complete South Africa or South Korea full coverage, claim Level A, add automated scheduled refresh, odds, payouts, predictions, tips, betting advice, or live coverage claims.

## Next work

After the remaining D-only scope is reduced, the next phase should move C/B records toward Level A through racecard/full race row parsers.
