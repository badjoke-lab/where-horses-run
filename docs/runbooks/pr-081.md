# PR-081: Country adapter template

## Summary

PR-081 adds a reusable country adapter template for the post-M3 / M4 country expansion phase.

The template standardizes how future country additions should move from source inventory to safe adapter or dry-run adapter, candidate JSON generation, candidate validation, approved bundle generation, cross-country validation, human review, and optional overlay promotion only after review.

## Files

- `docs/runbooks/country-adapter-template.md` — reusable M4 country adapter template and safety checklist.
- `scripts/check-country-adapter-template.mjs` — validation gate for the template, PR runbook, adapter contract, safety policy, and no new live-fetch implementation in the template files.
- `docs/runbooks/pr-081.md` — this PR runbook.
- `src/lib/timetable/country-adapter-template.ts` — build-safe helper for constructing an empty, review-only country adapter template result without fetching or parsing source pages.
- `package.json` — adds the `validate:country-adapter-template` script and runs it in `npm run check` after `validate:m3-v0-release-gate`.

## User-facing outcome: documentation/template only

This PR is documentation/template only.

It does not add South Korea, Turkey, Morocco, or any other new country data. It does not change public timetable coverage, UI behavior, generated overlays, or user-facing country pages.

## Safety boundary

The template preserves the safe source adapter policy:

```ts
storeSourceBody: false
storeRawMarkup: false
publishWithoutReview: false
allowedOutput: 'meeting_level_only'
```

This PR does not add:

- live fetch
- source page parsing
- raw source body storage
- raw markup storage
- racecards
- odds
- results
- payouts
- predictions
- tips
- public overlay replacement
- UI changes
- new dependencies

Official source links remain final confirmation, and partial coverage must remain clearly labeled.

## M4 usage

Future M4 country PRs should use the country adapter template in this order:

1. Source inventory.
2. Safe source adapter or dry-run adapter.
3. Candidate JSON generation.
4. Candidate validator.
5. Approved bundle generation.
6. Cross-country validation.
7. Human review.
8. Optional overlay promotion only after review.

Season gaps are allowed. If a future active-window bundle has no safe active-window meetings, it may use `records: []`, but it must clearly state season gap and no active-window meetings, and it must not be described as active public coverage.

## Validation commands

Run these commands before merging:

```bash
node scripts/check-country-adapter-template.mjs
npm run build
npm run check
```
