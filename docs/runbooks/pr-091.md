# PR-091: Add major-country sample acquisition fixtures

## Summary

PR-091 moves the major-country acquisition work from PR-090 source grouping to concrete sample acquisition targets. It defines static sample fixture targets for each acquisition group without adding live fetch runtime, parser implementation, raw source body storage, generated timetable records, odds, payouts, predictions, or tips.

## Files

- `data/static/major-country-sample-acquisition-fixtures.json` — static sample acquisition target definitions for every PR-090 group.
- `scripts/check-major-country-sample-acquisition-fixtures.mjs` — validator for country/group coverage, target shape, source linkage, annual/rolling separation, timing target coverage, Singapore legacy handling, and prohibited runtime/parser/timetable/product additions.
- `docs/runbooks/pr-091.md` — this runbook.
- `package.json` — adds `validate:major-country-sample-acquisition-fixtures` and runs it in `npm run check` immediately after `validate:major-country-acquisition-source-groups`.

## Why this PR exists

PR-088 established major-country racing inventory data, PR-089 added the country acquisition test matrix, and PR-090 grouped concrete acquisition sources. PR-091 is the next static planning layer: it identifies exact sample targets that PR-092 can turn into sample output snapshots.

This PR intentionally remains static. The fixture file records which source URL or derived racecard link should be sampled, the target grain, the parser target category, expected normalized fields, and whether the target is annual, rolling, first-race, per-race, meeting-date, or legacy inventory.

## Annual vs rolling sample handling

Annual fixture targets and rolling fixture/racecard targets are separate records. Annual samples are coarse season or calendar evidence and are not treated as final race-time truth. Rolling samples are active-window targets for date-level updates, racecard availability, declarations, or programme detail.

For groups with both annual and rolling PR-090 sources, the fixture file includes both `annual_fixture` and `rolling_fixture` records. For groups whose PR-090 source group only exposes rolling/day-level evidence, the fixture file includes a `rolling_fixture` record rather than inventing an annual target.

## First-race and per-race sample target handling

Every non-legacy group includes:

- at least one `annual_fixture` or `rolling_fixture` target;
- one `first_race_time` target for meeting-level first advertised start time; and
- one `per_race_time` target for race-level advertised start times.

These are sample targets only. They document where PR-092 should snapshot expected output shapes, but they do not fetch live pages or implement parsing.

## Singapore legacy handling

Singapore is handled as a legacy-only acquisition group. Active racing ended in Singapore, so the fixture file includes only one `racecourse_inventory` sample target for `singapore-turf-club-legacy`.

Singapore intentionally has no `first_race_time` target and no `per_race_time` target. No active timetable target is generated for Singapore.

## What this PR does

- Adds static sample acquisition fixture metadata for every PR-090 group.
- Links every non-legacy sample source URL to the matching PR-090 group source arrays, or to an explicit `derived:<source_id>` racecard link from that group.
- Keeps annual fixture samples separate from rolling/racecard samples.
- Defines expected normalized field names for future snapshots: `racecourse`, `meeting_date`, `first_race_time`, and `races[]`.
- Adds validation coverage and wires it into `npm run check`.

## What this PR does not do

- No live fetch runtime.
- No parser implementation.
- No generated timetable records.
- No raw source body storage.
- No odds support.
- No payouts support.
- No predictions support.
- No tips support.
- No new dependencies.
