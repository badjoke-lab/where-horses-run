# PR-076 — UAE ERA candidate generator

## Summary

This PR adds a dry-run UAE ERA candidate generator.

The generator reads only existing safe UAE timetable seed data and writes a reviewable candidate file. It does not live fetch, parse source pages, store source response bodies, or publish candidate records as public coverage.

- `scripts/check-uae-era-candidate-generator.mjs`

- `package.json`

## User-facing outcome

No direct UI change.

This PR prepares a UAE ERA candidate-generation path for review workflows without changing the public timetable overlay or country pages.

## Safety boundary

The UAE ERA generator is dry-run only.

This PR does not add live fetch.
This PR does not add source page parsing.
This PR does not store source response bodies.
This PR does not publish candidate records as coverage.
This PR does not replace the public overlay.
This PR does not add UI changes or new dependencies.

Generated records, when present, remain:

```text
status: candidate
review_status: needs_review
extraction_method: adapter_dry_run
```

## Season gap handling

UAE racing can be outside the current 30-day active window.

The generator uses the safe generated timetable window and includes only UAE ERA meetings that fall inside that window. Existing out-of-window UAE seed meetings are preserved as historical seed context, but they are not converted into current active-window candidate coverage.

When no safe active-window UAE ERA meetings are available, the generated candidate file keeps `records` empty and explicitly labels the output as a season gap / no active-window meeting state in review text and notes. This avoids pretending older seed meetings are current coverage.
