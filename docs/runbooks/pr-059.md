# PR-059 — Seed audit labels and coverage status UI

## Summary

This PR exposes the country racing inventory gate on English and Japanese country pages.

It does not add new timetable records. It makes the current limitations visible instead of letting seed data look like completed coverage.

## User-facing outcome

Country pages now show:

- overall coverage status
- current seed state
- racing systems
- system-level coverage status
- inventoried racecourse count
- official source candidates
- active 30-day record count
- out-of-window record count
- prohibited coverage claims

## Why this matters

The previous state made it too easy to mistake thin seed data for usable country coverage.

This PR makes the following visible:

- Japan is not JRA-only.
- Japan is split into JRA / NAR / Banei.
- NAR and Banei are still not started.
- Hong Kong remains initial seed only.
- UAE has records, but they are outside the active 30-day window.

## Safety boundary

This PR adds no live fetch, parser, source page fetching, raw HTML storage, response body storage, racecards, entries, horse names, jockey names, odds, results, payouts, prediction content, or tips.

## Validation

Run:

```sh
npm run validate:seed-audit-labels-ui
npm run validate:country-racing-inventory-gate
npm run check
```

Use Node v22.22.2 or compatible for full Astro build.
