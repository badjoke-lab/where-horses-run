## GitHub QA gate

The repository must verify:

- all sixteen built routes exist
- one h1 per route
- page-specific canonical links
- reciprocal English/Japanese hreflang links
- language switching to the corresponding country route
- reviewed official source links are rendered
- rank C pages do not render start-time or timezone meeting columns
- rank A pages do not exceed the reviewed A boundary
- no embedded video or `Watch here` label
- empty meeting data is not described as absence of racing
- runtime remains profile-v2 only
- entries remain at least `profile_ready` until the rendered preview passes

## CI ownership

`Check` is the only workflow that installs dependencies and runs the production build for a pull-request head. It then runs publication validators for entries 01–12, 13–20, and 21–28 against the same `dist` output.

The country-profile workflow is validator-only. The former separate publication build workflows for entries 01–12 and 13–20 are removed to prevent duplicate full builds.

## Preview gate

After GitHub QA is green:

1. Create or move one final `preview-*` branch to the reviewed PR head.
2. Allow exactly one Cloudflare preview deployment.
3. Review representative A and C pages in English and Japanese, plus responsive layout, source links, language switching, and empty states.
4. Record the preview result in the PR.
5. Only after the preview passes, advance entries 21–28 to `published`, set `qa_status=passed`, add `page_published_at`, and merge.
