# PR-114 — Equibase full racecard Level A pattern

## Purpose

PR-114 starts the C/B-to-A phase after the D-only cleanup completed in PR-113.

The goal is to establish the first source-family pattern for promoting fixture-backed B records into Level A records using full race rows.

## Scope

Included:

- `data/fixtures/timetable/us-equibase-racecard-full.html`
- `scripts/timetable/parsers/equibase-full-racecard.mjs`
- `scripts/generate-pr-114-equibase-level-a.mjs`
- `scripts/check-pr-114-equibase-level-a.mjs`

## Level A rule

A record can claim Level A only when it contains:

- country
- group/system
- racecourse
- meeting date
- first race time
- full race rows
- each race number
- each race time
- source trace
- source capture date
- last checked date

The validator also requires `first_race_time` to match race 1 time.

## Source family

This PR uses Equibase Thoroughbred as the first Level A pattern because PR-113 already created Equibase B-level records with real racecourse names.

## Non-goals

This PR does not add live fetch, complete United States full coverage, cover USTA or AQHA Level A, add automated scheduled refresh, odds, payouts, predictions, tips, betting advice, or live coverage claims.

## Next work

After this pattern is accepted, PR-115 and later can apply the same Level A approach to more source families and countries.
