# PR-068 — Japan NAR candidate generator

## Summary

This PR adds the Japan NAR candidate generator.

The generator is dry-run only. It reads the existing safe NAR meeting-level timetable overlay and writes a reviewable candidate file.

## Files

- `scripts/generate-japan-nar-candidates.mjs`
- `data/candidates/japan-nar-candidates.json`
- `scripts/check-japan-nar-candidate-generator.mjs`
- `package.json`
- `docs/runbooks/pr-068.md`

## User-facing outcome

No direct UI change.

This PR moves NAR data handling away from pure hand-edited overlay records and toward candidate generation.

## Safety boundary

This PR does not add live fetch.
This PR does not add source page parsing.
This PR does not store source response bodies.
This PR does not publish candidate records as coverage.
This PR excludes Banei records from NAR output.

The candidate file remains:

```text
review_status: needs_review
status: candidate
extraction_method: adapter_dry_run
```

## Validation

Run:

```sh
npm run generate:japan-nar-candidates
npm run validate:japan-nar-candidate-generator
npm run build
```

The validator also checks that the generated candidate output is not stale.
