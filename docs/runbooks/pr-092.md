# PR-092: Add major-country sample acquisition output snapshots

## Summary

PR-092 adds static sample acquisition output snapshots for every PR-091 sample fixture. The snapshots define the normalized output shape future parser contracts should produce while deliberately avoiding live source fetches, parser implementation, raw source storage, generated live timetable records, odds, payouts, predictions, and tips.

## Files

- `data/static/major-country-sample-acquisition-output-snapshots.json` — static expected normalized output snapshots linked one-for-one to the PR-091 sample acquisition fixtures.
- `scripts/check-major-country-sample-acquisition-output-snapshots.mjs` — validator for snapshot coverage, fixture linkage, placeholder-only normalized output shape, Singapore legacy behavior, and prohibited runtime/parser/product additions.
- `docs/runbooks/pr-092.md` — this runbook.
- `package.json` — adds `validate:major-country-sample-acquisition-output-snapshots` and runs it in `npm run check` immediately after `validate:major-country-sample-acquisition-fixtures`.

## Why this PR exists

PR-088 introduced major-country racing inventory data. PR-089 added the acquisition test matrix. PR-090 grouped acquisition sources. PR-091 selected concrete sample fixture targets from those groups.

PR-092 is the next static contract layer. It records the normalized output shape expected from each sample fixture so PR-093 can define parser contracts and normalization rules against an explicit snapshot shape. This PR does not claim that any live page was fetched or parsed; the snapshots are static placeholders that describe field shape only.

## Relationship to PR-091 sample fixtures

Every output snapshot is keyed back to a PR-091 fixture by:

- `country_id`
- `group_id`
- `sample_target_type`
- `sample_source_url`
- `parser_target`

The snapshot `annual_vs_rolling_rule` is copied from the matching PR-091 fixture, and `source_trace` records the fixture URL, fixture target role, and acquisition group ID. This keeps annual fixture, rolling fixture, first-race-time, per-race-time, and legacy inventory expectations separate.

## Normalized output shape

For each non-legacy sample fixture, `snapshot_kind` is `normalized_expected_output` and `snapshot_status` is `static_expected_shape_only`.

The placeholder normalized output shape is:

```json
{
  "racecourse": "<racecourse>",
  "meeting_date": "<meeting_date>",
  "first_race_time": "<first_race_time>",
  "races": [
    {
      "race_number": "<race_number>",
      "race_time": "<race_time>",
      "race_name_or_label": "<race_name_or_label>"
    }
  ]
}
```

The single `races[]` object is a shape placeholder only. It is not a generated record, live race row, racecard scrape, or timetable seed.

## Singapore legacy handling

Singapore is represented by exactly one legacy inventory snapshot for `singapore-turf-club-legacy`.

Singapore intentionally has:

- no active timetable output;
- no `first_race_time` output;
- no per-race `races[]` rows;
- `snapshot_status` set to `legacy_inventory_only`.

The Singapore snapshot exists only to preserve legacy inventory evidence for an inactive racing jurisdiction.

## What this PR does

- Adds static expected normalized output snapshots for every PR-091 sample fixture.
- Covers all 13 major-country scope countries.
- Preserves one-for-one linkage from output snapshot records to fixture source fields.
- Defines placeholder-only normalized fields for racecourse, meeting date, first race time, and per-race row shape.
- Keeps Singapore legacy inventory separate from active timetable output.
- Adds validation coverage and wires it into `npm run check`.

## What this PR does not do

- No live fetch runtime.
- No parser implementation.
- No generated live timetable records.
- No raw source body storage.
- No odds support.
- No payouts support.
- No predictions support.
- No tips support.
- No new dependencies.
