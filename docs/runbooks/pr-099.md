# PR-099 — Add preview timetable UI

## Summary

PR-099 makes the first static manual timetable samples visible on the website. It adds a user-visible preview page for the PR-098 preview data and keeps the scope deliberately static, transparent, and manually sourced.

PR-099 does not provide live coverage. PR-099 does not provide full country coverage. No odds, payouts, predictions, or tips are in scope.

## Files

- `src/pages/major-countries/preview-timetable.astro` — user-visible preview page for the first static manual timetable samples.
- `src/components/PreviewTimetableSamples.astro` — reusable mobile-readable sample-card renderer.
- `src/pages/countries/index.astro` — link from the countries section to the preview page.
- `data/static/preview-timetable-samples-batch-001.json` — static PR-098 preview timetable sample source used by the page.
- `scripts/check-preview-timetable-ui.mjs` — validator for the PR-099 UI scope.
- `package.json` — adds `validate:preview-timetable-ui` and includes it in `npm run check` immediately after `validate:preview-timetable-samples-batch-001`.
- `docs/runbooks/pr-099.md` — this operational note.

## What PR-099 adds

PR-099 adds a page at `/major-countries/preview-timetable/` that imports `data/static/preview-timetable-samples-batch-001.json` and renders the six current sample records.

The page includes:

- The title `Major Country Timetable Preview`.
- A plain-language notice that the page is a static manual sample preview.
- A plain-language notice that the page is not live coverage.
- A plain-language notice that full country coverage is not complete yet.
- A summary of preview sample count, represented countries, represented systems, and represented source capture dates.
- Per-sample cards with country, system, racecourse, meeting date, first race time, race list, official source link, source capture date, status label, and user notice.

## User-visible behavior

Users can open one page and see the current six static sample timetable records. The page is linked from the countries index so it is easy to find without knowing the route in advance.

Users should immediately understand that:

- The data is a static manual sample preview.
- The data is not live coverage.
- Full country coverage is not complete yet.
- Each sample is based on manually checked official/approved source pages.
- Each sample links to an official source URL for review and final confirmation.

The sample cards are designed to remain readable on mobile through a responsive grid that collapses to one column.

## Operational behavior

Operationally, PR-099 only renders existing static data. It does not schedule fetches, probe sources, store raw source bodies, run parsers, scrape websites, or generate live timetable records.

The new validator checks that:

- The preview page exists.
- The preview page references `preview-timetable-samples-batch-001.json`.
- The required user-facing title and scope notices are present.
- The UI references all required preview record display fields.
- The preview JSON contains at least the six PR-098 groups.
- The PR-099 UI does not add live fetch runtime, parser implementation, scraping dependencies, raw source body/html storage, generated live timetable records, odds, payouts, predictions, or tips.
- No new dependencies are added.

## What this PR does not do

PR-099 does not add:

- Live fetch runtime.
- Parser implementation.
- Scraping dependency.
- Raw source body or HTML storage.
- Generated live timetable records.
- Odds.
- Payouts.
- Predictions.
- Tips.
- Full country coverage.
