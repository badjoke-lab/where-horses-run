# PR-061 — Japan source-record split for NAR and Banei

## Summary

This PR splits Japan source records beyond the original JRA source.

It adds link-only official source records for:

- NAR home
- NAR racecourse guide
- Banei Tokachi home
- Banei monthly schedule source candidate

## User-facing outcome

Japan source routing can now distinguish:

- JRA
- NAR
- NAR racecourse guide
- Banei
- Banei schedule source candidate

This makes the source layer match the Japan inventory added in PR-060.

## Important limitation

This PR does not add timetable records.

It does not make Japan covered.
It does not make NAR covered.
It does not make Banei covered.
It does not add live fetch.
It does not add parser behavior.

All new records remain:

```text
link_only
alpha_link_first
live fetching requires separate source-specific review
```

## Safety boundary

This PR adds no source page fetching, raw HTML storage, response body storage, racecards, entries, horse names, jockey names, odds, results, payouts, prediction content, or tips.

## Validation

Run:

```sh
npm run validate:japan-source-record-split

npm run validate:country-racing-inventory-gate
npm run check
```

Use Node v22.22.2 or compatible for full Astro build.
