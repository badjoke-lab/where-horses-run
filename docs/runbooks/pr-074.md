# PR-074 — Hong Kong active-window refresh via generator

## Summary

This PR adds a deterministic reviewed Hong Kong active-window candidate bundle generator.

It reads the existing HKJC candidate file and writes an approved, reviewable candidate bundle for the Hong Kong active-window refresh path.

## Files

- `scripts/generate-hong-kong-active-window-approved-candidates.mjs`
- `scripts/check-hong-kong-active-window-approved-candidates.mjs`
- `data/candidates/hong-kong-active-window-approved-candidates.json`
- `docs/runbooks/pr-074.md`

## User-facing outcome

No direct UI change.

This PR prepares the generator-based refresh path for Hong Kong active-window data without directly replacing the public overlay yet.

## Safety boundary

This PR does not add live fetch.
This PR does not add source page parsing.
This PR does not store raw source bodies.
This PR does not store racecard, odds, results, payouts, predictions, tips, entries, horse names, jockey names, or trainer names.
This PR does not directly publish a refreshed overlay.
This PR does not replace the public overlay.
This PR does not add UI changes or new dependencies.

The bundle is generated from the existing safe HKJC candidate file:

- `data/candidates/hong-kong-hkjc-candidates.json`

The generated records are marked:

```text
status: source-reviewed
review_status: approved
```

## Validation commands

Run:

```sh
node scripts/generate-hong-kong-active-window-approved-candidates.mjs
node scripts/check-hong-kong-active-window-approved-candidates.mjs
npm run build
npm run check
```
