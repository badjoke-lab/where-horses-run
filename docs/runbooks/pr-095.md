# PR-095 — Major-country sample extraction audit matrix

## Summary

PR-095 adds a static audit matrix that ties the major-country acquisition work from PR-088 through PR-094 into one traceability view. The matrix is grouped by country and PR-090 acquisition group, then records whether each group has inventory, acquisition-test coverage, source-group definition, sample fixtures, normalized output snapshots, parser contracts, and sample extraction snapshots.

## Files

- `data/static/major-country-sample-extraction-audit-matrix.json` — static audit matrix for all major-country acquisition groups.
- `scripts/check-major-country-sample-extraction-audit-matrix.mjs` — validator for traceability, counts, status rules, and prohibited runtime/data additions.
- `docs/runbooks/pr-095.md` — this runbook.
- `package.json` — adds `validate:major-country-sample-extraction-audit-matrix` and wires it into `npm run check` after the PR-094 snapshot validator.

## Relationship to PR-088 through PR-094

The audit matrix references the prior static artifacts directly:

| PR | Static artifact | Audit role |
| --- | --- | --- |
| PR-088 | `data/static/major-country-racing-inventory.json` | Confirms the country inventory exists before group traceability is marked ready. |
| PR-089 | `data/static/major-country-acquisition-test-matrix.json` | Confirms country-level acquisition-test coverage exists. |
| PR-090 | `data/static/major-country-acquisition-source-groups.json` | Defines every acquisition group that must appear in the audit matrix. |
| PR-091 | `data/static/major-country-sample-acquisition-fixtures.json` | Supplies fixture counts by group and sample target type. |
| PR-092 | `data/static/major-country-sample-acquisition-output-snapshots.json` | Confirms every PR-091 fixture has a normalized output snapshot. |
| PR-093 | `data/static/major-country-parser-contracts.json` | Confirms every group has a static parser contract. |
| PR-094 | `data/static/major-country-sample-extraction-snapshots.json` | Supplies extraction snapshot counts and confirms every PR-091 fixture has a static extraction example. |

## Audit dimensions

Each audit entry records:

- `country_id`
- `group_id`
- `update_model`
- `inventory_present`
- `acquisition_matrix_present`
- `source_group_present`
- `sample_fixtures_present`
- `output_snapshots_present`
- `parser_contract_present`
- `extraction_snapshots_present`
- `annual_fixture_sample_count`
- `rolling_fixture_sample_count`
- `first_race_time_sample_count` for non-legacy groups only
- `per_race_time_sample_count` for non-legacy groups only
- `extraction_snapshot_count`
- `traceability_status`
- `remaining_before_live_extraction`

## Counts and traceability status

The validator derives counts from PR-091 and PR-094 and compares them with the audit matrix:

- `annual_fixture_sample_count` must match PR-091 annual fixture samples for the group.
- `rolling_fixture_sample_count` must match PR-091 rolling fixture samples for the group.
- `first_race_time_sample_count` must match PR-091 first-race-time samples and be greater than zero for each non-legacy group.
- `per_race_time_sample_count` must match PR-091 per-race-time samples and be greater than zero for each non-legacy group.
- `extraction_snapshot_count` must match both the number of PR-094 extraction snapshots and the number of PR-091 fixtures for the group.
- Non-legacy groups must use `ready_for_manual_source_snapshot`.

## Singapore legacy handling

Singapore is the only `legacy_inventory_only` entry. Its audit entry intentionally differs from active non-legacy groups:

- `traceability_status` is `legacy_inventory_only`.
- `first_race_time_sample_count` is omitted.
- `per_race_time_sample_count` is omitted.
- `extraction_snapshot_count` is exactly `1`.
- The remaining step states that it stays legacy-only until racing resumes or an approved successor source exists.

## What this PR does

- Adds a static traceability matrix for every major-country acquisition group.
- Verifies every PR-090 group is represented.
- Verifies every country from the 13-country major-country scope is represented.
- Links PR-088 through PR-094 static artifacts into one audit view.
- Verifies PR-091 fixture counts and PR-094 extraction snapshot counts match.
- Adds a validation command and includes it in `npm run check` after the PR-094 extraction snapshot validation.

## What this PR does not do

This PR does not add:

- live fetch runtime
- parser implementation
- scraping dependency
- raw source body or HTML storage
- generated live timetable records
- odds
- payouts
- predictions
- tips
- new dependencies
