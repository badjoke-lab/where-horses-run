# PR-058 — Country racing inventory and seed validity gate

## Summary

This PR adds a country racing inventory gate before any thin timetable seed can be treated as country coverage.

The inventory records the known racing systems, official source candidates, racecourse scope, current seed state, and forbidden coverage claims for the current top countries:

- Japan
- Hong Kong
- United Arab Emirates

## Why this exists

The current timetable records are only thin source-reviewed seeds:

- Hong Kong has an initial HKJC seed, not complete Hong Kong coverage.
- Japan has only four JRA records for Tokyo and Kyoto, not full Japan coverage.
- UAE has source-reviewed records that are outside the active 30-day calendar window.

This PR prevents those records from being treated as complete or broad coverage.

## User-facing outcome

The site can now distinguish the following states in data and future UI work:

- not_started
- initial_seed_only
- partial
- out_of_window_seed
- broad
- complete

No current country is allowed to use `complete`.

## Inventory scope

### Japan

Japan is split into:

- JRA / central racing
- NAR / local government racing
- Banei / Obihiro

JRA includes the ten in-scope JRA racecourses:

- Sapporo
- Hakodate
- Fukushima
- Niigata
- Tokyo
- Nakayama
- Chukyo
- Kyoto
- Hanshin
- Kokura

Japan remains `initial_seed_only` because only Tokyo and Kyoto have seed records. NAR and Banei remain `not_started`.

### Hong Kong

Hong Kong is mapped as HKJC with:

- Sha Tin
- Happy Valley

Hong Kong remains `initial_seed_only` until a target-window fixture completion review exists.

### United Arab Emirates

UAE is mapped to Emirates Racing / ERA sources and the currently seen seed racecourses:

- Meydan
- Al Ain
- Abu Dhabi Turf Club

UAE remains `out_of_window_seed` because current records are before the active 30-day window derived from `generated_at`.

## Safety boundary

This PR adds no new timetable records.

It does not add:

- live fetch
- parser
- source page fetching
- raw HTML storage
- response body storage
- racecards
- entries
- horse names
- jockey names
- odds
- results
- payouts
- prediction content
- tips

## Validation

Run:

```sh
npm run validate:country-racing-inventory-gate
npm run check
```

Use Node v22.22.2 or compatible for the full Astro build.
