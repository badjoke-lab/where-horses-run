# PR-065 — Timetable candidate schema / review format

## Summary

This PR starts the move away from hand-written timetable seed files by defining the shared candidate format used before overlay promotion.

## Files

- `data/generated/timetable-candidates.schema.json`
- `data/generated/timetable-candidates.sample.json`
- `docs/runbooks/timetable-candidate-review-format.md`
- `scripts/check-timetable-candidate-schema.mjs`

## User-facing outcome

No direct UI change.

This PR creates the data contract required before JRA, NAR, Banei, HKJC, ERA, and later country adapters can generate reviewable timetable candidates.

## Rules fixed by this PR

- Candidate files are review inputs, not public coverage.
- Only approved candidate records may be promoted to overlay files.
- Rejected records must never be promoted.
- Candidate generation must not automatically publish site-facing data.
- Meeting-level safe fields only are allowed.
- Raw HTML, response bodies, racecards, entries, odds, results, payouts, predictions, and tips are forbidden.

## Validation

Run:

```sh
node scripts/check-timetable-candidate-schema.mjs
```

If package wiring is present:

```sh
npm run validate:timetable-candidate-schema
npm run check
```
