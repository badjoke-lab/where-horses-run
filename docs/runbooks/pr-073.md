# PR-073 — Hong Kong HKJC candidate generator

## Summary

This PR adds the Hong Kong HKJC candidate generator.

The generator is dry-run only. It reads the existing safe HKJC meeting-level timetable seed and writes a reviewable candidate file.

## Files

- `scripts/generate-hong-kong-hkjc-candidates.mjs`
- `data/candidates/hong-kong-hkjc-candidates.json`
- `scripts/check-hong-kong-hkjc-candidate-generator.mjs`
- `docs/runbooks/pr-073.md`

## User-facing outcome

No direct UI change.

This PR moves Hong Kong HKJC data handling away from pure hand-edited seed records and toward candidate generation.

## Safety boundary

This PR does not add live fetch.
This PR does not add source page parsing.
This PR does not store source response bodies.
This PR does not publish candidate records as coverage.

The candidate file remains:

```text
review_status: needs_review
status: candidate
extraction_method: adapter_dry_run
```

## Normalization

The existing seed has some Happy Valley records without `racecourse_id`.

This PR normalizes those records to:

```text
happy-valley-racecourse
```

Sha Tin records remain:

```text
sha-tin-racecourse
```

## Validation

Run:

```sh
node scripts/generate-hong-kong-hkjc-candidates.mjs
node scripts/check-hong-kong-hkjc-candidate-generator.mjs
npm run build
```

Package wiring is intentionally not included in this PR.
