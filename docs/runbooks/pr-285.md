# PR-285 — Add the source-test handoff boundary

## Summary

This PR adds the remote-first handoff layer used when a source test cannot be
completed remotely.

It provides:

- a versioned JSON schema shell,
- an allowlist-based exporter,
- split validation for identities, acquisition metadata, meeting data,
  artifact digests, ranks and field names,
- valid, expected and invalid fixtures,
- a dedicated GitHub Actions workflow,
- a concise operational runbook.

## Safety boundary

The exporter does not open captured source files. The acquisition step must
supply artifact SHA-256 digests and byte sizes in its input manifest. The
exporter then reconstructs a new object from approved fields and discards extra
working fields.

Only generated handoff JSON is used by later remote work.

## Validation

The dedicated workflow:

1. exports the reference input,
2. validates the generated handoff,
3. compares it byte-for-byte with the expected fixture,
4. verifies that an invalid digest and rank are rejected.
