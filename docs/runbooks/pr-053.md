# PR-053 — Tomorrow page using safe timetable data

## Summary

The Tomorrow page now uses `siteData.generated.timetables` from `data/generated/timetables.json` to list safe timetable records for the day after the generated timetable date.

## Empty state

If no records match tomorrow, the page shows a clear empty state instead of implying that verified data exists:

- English: `No safe timetable records are listed for tomorrow yet.`
- Japanese: `明日の安全な開催時刻レコードはまだありません。`

Both pages direct visitors to use official source links for final confirmation.

## Safety boundary

Live fetching remains disabled. This PR does not add a parser, source page fetcher, or any source response storage.

The Tomorrow pages only prepare the safe timetable fields needed for display: country, racecourse, date, start time, timezone, racing type, official source, last checked, status, confidence, and notes.

The Tomorrow pages do not display racecard, entries, odds, results, payout, prediction, or tip content.

## Validation

Run these checks before release:

```sh
npm run validate:tomorrow-timetable-ui
npm run validate:timetable-schema-v0
npm run check
```

Use Node v22.22.2 or another Astro-compatible Node version.
