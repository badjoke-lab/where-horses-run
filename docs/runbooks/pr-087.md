# PR-087 — Major countries v0 scope lock and data schema

## Summary

PR-087 locks the major-countries v0 country scope, adds a machine-readable data schema for the scope, and adds a validator so subsequent PRs can complete countries in batches. The target is major-countries v0 completion by PR-096 at the latest.

## Files

- `docs/runbooks/major-countries-v0-scope.md`
- `data/static/major-countries-v0-scope.json`
- `scripts/check-major-countries-v0-scope.mjs`
- `docs/runbooks/pr-087.md`
- `package.json`

## Why the workflow is changing

The previous per-country micro-PR workflow is too slow for major-countries v0. Japan alone required many PRs and remains partial. PR-087 pauses per-country micro PRs, gap-report-only PRs, and source-evidence-only PRs unless evidence is bundled with data/schema/page changes.

The new workflow batches authority records, racecourse inventory, official calendar links, and safe active-window data across groups of scoped countries. This makes it possible to finish a usable major-country index within PR-087 through PR-096.

## Major countries scope

The major-countries v0 scope is:

- Japan
- Hong Kong
- United Arab Emirates
- United Kingdom
- Ireland
- France
- United States
- Canada
- Australia
- New Zealand
- South Africa
- South Korea
- Singapore

Japan/Hong Kong/UAE existing data remains partial. Major-countries v0 means usable major-country index, not full worldwide timetable completion. Complete global coverage is not claimed, and this PR does not claim every major country has active-window timetables.

## 10-PR cap

PR-087 through PR-096 are capped for major-countries v0 completion. The cap is intentionally explicit so future work batches data and validation changes instead of returning to one country or one system per PR.

## What this does not do

- Does not add new timetable records.
- Does not add live fetch runtime.
- Does not add a source parser.
- Does not add racecards.
- Does not add odds.
- Does not add results.
- Does not add payouts.
- Does not add predictions.
- Does not add tips.
- Does not make public complete coverage claims.
- Does not add dependencies.
