# PR-100 — Add remaining source snapshots and preview samples

## Summary

PR-100 expands manual official-source confirmations to the remaining active major-country acquisition groups that were not covered by PR-098. It adds a second static snapshot batch, normalized static timetable samples, and preview-ready samples for the active acquisition groups outside Japan, Hong Kong, the United Arab Emirates, and South Korea.

PR-100 does not provide live coverage. It does not guarantee full country, date, racecourse, annual, or per-race coverage. The records are static manual samples captured from official or approved source pages and are intended to make the next preview-index PR possible without adding runtime fetching, parser implementation, scraping dependencies, or generated live timetable records.

No odds, payouts, predictions, or tips are in scope.

## Files

- `data/static/manual-source-snapshots-batch-002.json` — manual official/approved source snapshot records for the remaining active major-country groups.
- `data/static/normalized-timetable-samples-batch-002.json` — normalized timetable sample records backed only by captured batch-002 snapshots.
- `data/static/preview-timetable-samples-batch-002.json` — display-ready sample records for a future combined preview index.
- `scripts/check-preview-timetable-samples-batch-002.mjs` — validator for batch-002 files, group coverage, preview readiness, and prohibited additions.
- `package.json` — adds `validate:preview-timetable-samples-batch-002` immediately after `validate:preview-timetable-ui` in the full check chain.
- `docs/runbooks/pr-100.md` — this runbook.

## Current schedule and current position

The major-country work is still in a staged static-confirmation phase. PR-098 supplied the first source-confirmed preview samples for six active groups. PR-099 made those first-batch samples visible at `/major-countries/preview-timetable/`.

PR-100 is the second static sample batch. It widens the preview sample pool to the rest of the active major-country acquisition scope, but it intentionally remains static and manually captured.

## What PR-099 completed

PR-099 added the user-facing `/major-countries/preview-timetable/` page for PR-098 batch-001 samples. It rendered the six existing preview records and made clear that those records were static manual samples, not live timetable coverage.

## What PR-100 adds

PR-100 adds batch-002 records for the remaining active major-country acquisition groups:

- Manual source snapshot records with official/approved source URLs, capture dates, local meeting context, first-race evidence, sampled per-race evidence, normalized sample data, source traces, reviewer notes, and status labels.
- Normalized timetable records derived only from captured batch-002 snapshots.
- Preview-ready display records derived from the normalized records.
- A validator that confirms group coverage, absence of Singapore active samples, batch-001/batch-002 merge readiness, and the no-live-fetch/no-parser/no-scraping/no-generated-live-records boundaries.

## User-visible meaning

Users can eventually see more representative static examples in the major-country preview area after a future UI/index PR wires batch 002 into the page. These samples show what official-source-backed timetable rows can look like for additional countries and systems.

The user-visible status remains limited:

- This is static manual sample coverage.
- This is not live coverage.
- Full country coverage is not complete yet.
- Full date and racecourse coverage is not complete yet.
- Some samples are partial static manual samples where only selected race rows are recorded.

## Operational meaning

Operationally, PR-100 provides curated static fixtures that can be used to validate preview indexing and display behavior. The PR does not create a runtime acquisition pipeline and does not parse official pages.

The batch-002 records should be treated as reviewable source-confirmation evidence, not as production timetable feeds.

## Captured vs needs_source_correction status

The batch-002 manual snapshot schema allows either `captured_static_sample` or `needs_source_correction` for each group. PR-100 records the remaining groups as `captured_static_sample` because each group has a manual official/approved source URL and sampled race-time evidence in this batch.

If a later review finds that a source page cannot support the recorded first-race or per-race evidence, the affected record should be changed to `needs_source_correction` and removed from normalized and preview sample files until corrected.

## Partial sample handling

Partial static manual samples are allowed when full per-race rows are not confirmed. A partial sample must clearly say it is partial in the normalized `user_visible_summary` and the preview `user_notice`.

PR-100 uses partial-sample language for groups where only selected official/approved race rows are recorded rather than complete meeting coverage. Partial samples are still not live coverage and do not imply complete country, date, or racecourse coverage.

## Singapore legacy handling

Singapore remains legacy/no active racing. PR-100 does not add active manual snapshots, normalized samples, or preview samples for `singapore/singapore-turf-club-legacy`.

Singapore should stay out of active preview sample indexes unless a future scope decision explicitly reclassifies it; this PR does not do that.

## What this PR does not do

- Guarantee complete country/date/racecourse coverage.
- Add live fetch runtime.
