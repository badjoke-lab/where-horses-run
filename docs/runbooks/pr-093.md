# PR-093: Major-country parser contracts

## Summary

PR-093 adds static parser contracts and normalization rules for the major-country acquisition pipeline. The contracts are based on the PR-090 acquisition groups, PR-091 sample fixture targets, and PR-092 normalized output snapshot shapes.

This PR is contract-only. It does not fetch live source pages, parse source pages, scrape websites, store raw source bodies, or generate live timetable records.

## Files

- `data/static/major-country-parser-contracts.json` — contract data for all 13 major-country scope countries and every PR-090 acquisition group.
- `scripts/check-major-country-parser-contracts.mjs` — validator for coverage, required fields, normalization rules, Singapore legacy handling, and prohibited runtime/data additions.
- `docs/runbooks/pr-093.md` — this runbook.
- `package.json` — adds `validate:major-country-parser-contracts` and includes it in `npm run check` immediately after the PR-092 snapshot validator.

## Why this PR exists

The major-country acquisition work now has:

1. Inventory scope from PR-088.
2. Acquisition test matrix from PR-089.
3. Source groups from PR-090.
4. Static sample acquisition fixtures from PR-091.
5. Expected normalized output snapshots from PR-092.

PR-093 defines the parser-facing contract that future sample extraction work must satisfy before any live runtime or parser implementation is introduced. This keeps the pipeline sequence explicit: define sources, define sample targets, define normalized output shapes, then define parser contracts.

## Relationship to PR-092 snapshots

PR-092 established the expected normalized output shape for static samples:

- `racecourse`
- `meeting_date`
- `first_race_time`
- `races[]`
- `races[].race_number`
- `races[].race_time`
- `races[].race_name_or_label`
- `source_trace`

PR-093 uses that shape as the required output contract for every non-legacy PR-090 acquisition group. It does not add generated rows. The `races[]` object remains a contract shape for future extraction snapshots, not a live timetable payload.

## Normalization contract

For every non-legacy acquisition group, the contract defines:

- `country_id`
- `group_id`
- `parser_target`
- `input_contract`
- `output_contract`
- `normalization_rules`
- `required_output_fields`
- `annual_vs_rolling_rule`
- `timezone_rule`
- `race_number_rule`
- `race_time_rule`
- `racecourse_name_rule`
- `meeting_date_rule`
- `failure_modes`
- `next_sample_extraction_step`

The normalized output contract is deliberately narrow. It covers meeting identity and timing only. Odds, payouts, predictions, and tips are out of scope.

## Annual vs rolling handling

Annual fixture data is not final timing truth. Annual fixture sources can identify season calendars, racecourse-date rows, or coarse meeting structure, but rolling fixture, racecard, entries, declarations, or daily race information sources can override annual timing when they identify the same country, group, racecourse, and meeting date.

The contract keeps annual and rolling source roles separate so future extraction can test whether timing changed between annual fixture inventory and active-window racecard sources.

## Timezone handling

Timezone handling must be source-country local time first. Future extraction should preserve the local source time and source-country meeting date during normalization. UTC conversion is a later derived step only after the normalized local record exists.

This prevents source-country evening or early-morning meetings from shifting dates during contract validation.

## Singapore legacy handling

Singapore has only the `singapore-turf-club-legacy` inventory contract. It intentionally has:

- No active timetable contract.
- No `first_race_time` contract.
- No per-race contract.
- No generated timetable output shape.

Singapore remains in the major-country scope as legacy inventory only because active racing has ceased.

## What this PR does

- Adds parser contract coverage for all 13 countries.
- Adds parser contract coverage for every PR-090 acquisition group.
- Defines required normalized output fields for non-legacy groups.
- Defines annual-vs-rolling, timezone, race number, race time, racecourse name, and meeting date rules.
- Adds validation to prevent accidental live runtime, parser implementation, generated timetable records, raw source body storage, or betting/tipping fields.

## What this PR does not do

- Does not add live fetch runtime.
- Does not add parser implementation.
- Does not add scraping.
- Does not store raw source bodies.
- Does not add generated live timetable records.
- Does not add odds.
- Does not add payouts.
- Does not add predictions.
- Does not add tips.
- Does not add dependencies.
