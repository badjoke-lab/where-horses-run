# PR-116 — AQHA Quarter Horse full racecard Level A pattern

## Purpose

PR-116 continues the C/B-to-A phase after PR-114 and PR-115 established Level A patterns for Equibase Thoroughbred and USTA Harness.

The goal is to add the same Level A pattern for the AQHA Quarter Horse source family.

## Scope

Included:

- `data/fixtures/timetable/us-aqha-racecard-full.html`
- `scripts/timetable/parsers/aqha-full-racecard.mjs`
- `scripts/generate-pr-116-aqha-level-a.mjs`
- `scripts/check-pr-116-aqha-level-a.mjs`

## Level A rule

A record can claim Level A only when it contains:

- country
- group/system
- racecourse
- meeting date
- first race time
- full race rows
- each race number
- each race time
- source trace
- source capture date
- last checked date

The validator also requires `first_race_time` to match race 1 time.

## Source family

This PR uses AQHA Quarter Horse. PR-113 had to use neutral course labels for this source family, so PR-116 keeps those neutral labels while establishing the Level A data shape.

## Non-goals

This PR does not add live fetch, complete United States full coverage, replace AQHA neutral course labels with real course names, add automated scheduled refresh, odds, payouts, predictions, tips, betting advice, or live coverage claims.

## Next work

After PR-116, the US source-family Level A patterns are aligned for Equibase, USTA, and AQHA. PR-117 can expand Level A coverage count or start applying the same full race rows approach to another country/source family.
