# PR-110 — Harness Australia parser-backed promotion

## Purpose

PR-110 is the first parser-backed promotion PR after the refresh skeleton in PR-109.

The goal is to stop treating Harness Australia as a D-only source target and prove that official Harness Australia page content can be parsed into promotion records.

## Scope

Included:

- Fixture snapshot from the official Harness Australia fields/current races page rail.
- Harness Australia parser for current race rows.
- Generator that converts the fixture into normalized promotion records.
- Validator that proves parser-backed D-to-C/B promotion output.
- Package check integration.

## Files added

- `data/fixtures/timetable/harness-australia-current-races.html`
- `scripts/timetable/parsers/harness-australia-fields.mjs`
- `scripts/generate-pr-110-harness-australia-promotions.mjs`
- `scripts/check-pr-110-harness-australia-promotion.mjs`

## Package script added

- `validate:pr-110-harness-australia-promotion`

Included in `npm run check`.

## Data level handling

This PR does not claim Level A.

The current races rail exposes racecourse names and first visible race times. That is enough to prove parser-backed promotion away from D, but not enough to claim all race times for a meeting.

The output can therefore produce:

- Level C when racecourse/meeting evidence exists but durable first-race treatment is incomplete.
- Level B when the first-race time is available.

Level A requires the meeting page parser and full race rows.

## User-facing meaning

Harness Australia should no longer be treated as source-only D in the long-term plan.

This PR proves that official Harness Australia content can produce usable timetable promotion records, but the next parser should still capture stable meeting calendar dates and full meeting pages.

## Operational meaning

The PR-109 skeleton is now being used by a real source family:

1. Keep a source-specific fixture.
2. Parse official page content.
3. Generate normalized promotion records.
4. Validate level and source trace.

## Non-goals

This PR does not add live fetch, complete Harness Australia full coverage, claim Level A, add automated scheduled refresh, odds, payouts, predictions, tips, betting advice, or live coverage claims.

## Next work

Next Harness Australia work should add a stable meeting calendar or meeting page parser so that:

- meeting dates are durable, not only rail-derived;
- all race rows can be captured;
- records can move from B to A where official full race rows are available.
