# PR-069 — Japan Banei candidate generator

## Summary

This PR adds the Japan Banei candidate generator.

The generator is dry-run only. It reads the existing safe Banei meeting-level timetable overlay and writes a reviewable candidate file.

## Files

- `scripts/generate-japan-banei-candidates.mjs`
- `data/candidates/japan-banei-candidates.json`
- `scripts/check-japan-banei-candidate-generator.mjs`
- `package.json`
- `docs/runbooks/pr-069.md`

## User-facing outcome

No direct UI change.

This PR moves Banei data handling away from pure hand-edited overlay records and toward candidate generation.

## Safety boundary

This PR does not add live fetch.
This PR does not add source page parsing.
This PR does not store source response bodies.
This PR does not publish candidate records as coverage.
This PR keeps Banei output separate from NAR output.

The candidate file remains:

```text
review_status: needs_review
status: candidate
extraction_method: adapter_dry_run
```

## Validation

Run:

```sh
npm run generate:japan-banei-candidates
npm run validate:japan-banei-candidate-generator
npm run build
```

The validator also checks that the generated candidate output is not stale.
