# PR-096: Major-country v0 completion gate

## Summary

PR-096 closes the PR-088 through PR-095 static acquisition-readiness chain for `major-countries-v0` and defines the next phase: manual static source snapshots from official or approved source pages.

This is not a completed public timetable product. It is only a static acquisition-readiness gate. Users still cannot see real major-country timetable data from these artifacts alone.

## What PR-096 completes

PR-096 marks the static preparation chain as complete for acquisition readiness only. It adds a completion gate that records:

- `static_acquisition_readiness_complete` status for `major-countries-v0`.
- The eight completed preparation PRs, PR-088 through PR-095.
- The eight completed static artifacts from the inventory, matrix, source-group, fixture, output, parser-contract, extraction-snapshot, and audit-matrix chain.
- The next phase, `manual_static_source_snapshots`.
- The next PR, PR-097, for the manual source snapshot schema and first batch plan.

## Current user-visible status

The public site does not yet show real major-country timetable data from this chain. The PR-096 gate explicitly says the repository is ready to start manual source snapshot work, but real major-country timetable data is not yet visible to users.

## Current operational status

Source groups, sample targets, parser contracts, placeholder extraction shapes, and audit traceability are prepared. Manual source snapshots, real normalized samples, UI display, and update operations are still not complete.

## What is explicitly not complete

The completion gate blocks any false claim that major-countries v0 already has a finished live timetable product. These items remain not complete:

- No live fetch runtime.
- No parser implementation.
- No real source snapshots captured yet.
- No real normalized timetable records generated yet.
- No public timetable UI for major-country data yet.
- No cron or scheduled update process.
- No raw source body storage.
- No generated live timetable records.
- No odds.
- No payouts.
- No predictions.
- No tips.

## Manual source snapshot plan

The manual source snapshot plan covers all 13 countries and all 25 PR-090 acquisition groups. For every non-legacy group, the plan requires:

- `manual_snapshot_required: true`.
- `snapshot_capture_mode: manual_static_source_snapshot`.
- PR-095 traceability status of `ready_for_manual_source_snapshot`.
- A user-visible goal explaining what this source must eventually prove for site users.
- A plain-language operator or racing-system scope.
- Snapshot source basis across inventory, acquisition matrix, source groups, sample fixtures, parser contracts, and extraction audit matrix.
- Minimum targets for racecourse inventory or meeting context, fixture context, first-race time, per-race time, official source URL, and source capture date.
- Requirements before live extraction: one verified manual static source snapshot, one normalized sample record, source URL trace, and confirmation that no odds/payouts/predictions/tips are included.

## Annual fixture vs rolling/racecard handling

Annual fixture pages are not final timing truth. Annual or season fixture material is only a coarse schedule source.

Rolling racecard, entries, declarations, or daily race information is required for first-race and per-race timing confirmation before any source is treated as final timing evidence.

## Singapore legacy handling

Singapore remains legacy/no active racing. The Singapore entry is `singapore-turf-club-legacy`, has `manual_snapshot_required: false`, and uses `snapshot_capture_mode: legacy_inventory_only`.

Singapore has no active timetable targets, no first-race or per-race timing requirements, and no live extraction path unless racing resumes or an approved successor source exists.

## Completion gate

`data/static/major-country-v0-completion-gate.json` records the static acquisition-readiness completion state. It does not assert that public timetable output exists. It explicitly lists the missing runtime, source confirmation, generated data, UI, scheduler, and excluded betting-adjacent categories.
