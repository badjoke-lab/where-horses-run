# PR-111 — Canada parser-backed promotion

## Purpose

PR-111 continues the 13-country D-to-C/B promotion work after PR-110.

The goal is to promote Canada records away from D-only source targets by using parser-backed official operator race-day fixtures for:

- Woodbine Thoroughbred
- Canadian Standardbred via Woodbine Mohawk Park official operator pages

## Scope

Included:

- Woodbine Racetrack fixture for thoroughbred race-day rows.
- Woodbine Mohawk Park fixture for standardbred race-day rows.
- Shared Canada Woodbine race-day parser.
- Generator that emits normalized promotion records.
- Validator that checks both Canadian groups produce Level B promotion records.
- Package check integration.

## Files added

- `data/fixtures/timetable/canada-woodbine-race-day.html`
- `data/fixtures/timetable/canada-woodbine-mohawk-race-day.html`
- `scripts/timetable/parsers/canada-woodbine-race-days.mjs`
- `scripts/generate-pr-111-canada-promotions.mjs`
- `scripts/check-pr-111-canada-promotion.mjs`

## Package script added

- `validate:pr-111-canada-promotion`

Included in `npm run check`.

## Data level handling

This PR does not claim Level A.

The race-day rows expose meeting date, racecourse, and first post time. That supports Level B promotion. Full race rows require a racecard parser before Level A can be claimed.

## Source handling

For standardbred Canada, the registry family remains `standardbred-canada`, but this PR uses Woodbine Mohawk Park as an official operator source for race-day rows.

This is acceptable as an operator source for the meeting rows, while Standardbred Canada remains a broader national source family to improve later.

## User-facing meaning

Canada no longer needs to remain D-only in the major-country scope.

The promoted rows can support display of:

- country
- system/group
- racecourse
- meeting date
- first post time
- official source link
- source capture date

## Operational meaning

PR-111 proves the same parser-backed promotion shape for Canada that PR-110 proved for Harness Australia:

1. Store a source-specific fixture.
2. Parse official operator page content.
3. Generate normalized promotion records.
4. Validate group, date, source, first post time, and promotion source trace.

## Non-goals

This PR does not add live fetch, complete Canada full coverage, claim Level A, add automated scheduled refresh, odds, payouts, predictions, tips, betting advice, or live coverage claims.

## Next work

Next Canada work should add full racecard parsers so Level B records can move to Level A when full race rows are available.

The next major-country D-to-C PR should target South Africa / South Korea or the United States, depending on source stability.
