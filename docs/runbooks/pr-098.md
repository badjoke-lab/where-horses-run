# PR-098: Add first source snapshots and preview timetable samples

## Summary

PR-098 is the first PR that records real manual official-source confirmations for the PR-097 first batch. It adds static source-snapshot records, normalized sample timetable records, and preview-ready sample data that a later UI PR can display without adding live fetching or parser runtime.

PR-098 also creates preview-ready static timetable sample data. Users still cannot browse complete live timetables from this PR alone.

## Files

- `data/static/manual-source-snapshots-batch-001.json` records the six manual official-source snapshots.
- `data/static/normalized-timetable-samples-batch-001.json` normalizes captured static samples into a common timetable shape.
- `data/static/preview-timetable-samples-batch-001.json` provides display-facing static preview records for the next UI PR.
- `scripts/check-preview-timetable-samples-batch-001.mjs` validates the batch, group coverage, non-placeholder evidence, preview notices, and prohibited additions.
- `package.json` adds `validate:preview-timetable-samples-batch-001` immediately after `validate:manual-source-snapshot-schema` in the full check chain.
- `docs/runbooks/pr-098.md` documents the batch scope and operating limits.

## Current schedule and current position

The project is moving from schema/planning work into real source confirmation. PR-097 selected the first six high-confidence acquisition groups for manual capture. PR-098 captures official-source evidence for those groups as static manual samples and prepares a small preview data layer.

This is still before live timetable support. The records are deliberately static, manually reviewed samples.

## What PR-097 completed

PR-097 established the first-batch manual source snapshot plan for:

- Japan / JRA
- Japan / NAR
- Japan / Banei
- Hong Kong / HKJC
- United Arab Emirates / ERA
- South Korea / KRA

It also kept Singapore out of the active snapshot batch because Singapore is currently treated as legacy/no active racing for this project.

## What PR-098 adds

PR-098 adds three connected data layers for the same first-batch groups:

1. Manual source snapshots with official-source trace fields and reviewer notes.
2. Normalized timetable sample records for groups with captured static evidence.
3. Preview-ready static timetable sample records with user-facing notices.

These are real manual source confirmations, not another abstract schema layer.

## User-visible meaning

A future UI can display sample timetable cards for the captured groups using static data. Each preview record states that it is a static manual sample, is not live coverage, and does not mean full country coverage is complete.

Users still cannot browse complete live timetables from this PR alone.

## Operational meaning

Maintainers now have a first concrete batch showing how official source evidence maps into normalized and preview-facing records. The validator protects the batch against:

- unexpected groups,
- Singapore active snapshot drift,
- placeholder evidence,
- normalized samples for source-correction records,
- missing static/manual/not-live/incomplete-coverage user notices,
- raw source body or HTML storage,
- live fetch runtime,
- parser implementation,
- scraping dependencies,
- betting-adjacent fields such as odds, payouts, predictions, or tips.

All six PR-097 target groups are captured as static manual samples in this batch:

- `japan/nar` — NAR Ohi sample, 2026-05-18.

- `hong-kong/hkjc` — HKJC Sha Tin sample, 2026-04-12.
- `united-arab-emirates/era` — ERA Meydan sample, 2026-03-28.

## Groups needing source correction, if any

No PR-098 first-batch group remains `pending_manual_capture`. This batch does not mark any of the six selected groups as `needs_source_correction` because each has at least one official-source-backed static sample suitable for the preview data layer.

Some samples are intentionally narrow where the captured official surface exposed only selected race times. Later PRs should broaden those sources rather than treating this batch as full timetable coverage.

## Static preview data

`preview-timetable-samples-batch-001.json` is the display-facing bridge for PR-099 or a later UI PR. It contains country/system display labels, racecourse/date/first-race display fields, selected display races, official source URLs, capture dates, and explicit user notices.

The preview file is static. It is not generated by a live fetch and does not imply full coverage.

## What this PR does not do

PR-098 does not:

- claim full 13-country completion,
- claim complete major-country timetable coverage,
- add live timetable support,
- add live fetch runtime,
- add parser implementation,
- add scraping dependencies,
- store raw HTML, raw page bodies, or source text,
- add generated live timetable records,
- add odds,
- add payouts,
- add predictions,
- add tips.
