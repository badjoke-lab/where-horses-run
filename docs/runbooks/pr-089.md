# PR-089: Major-country racecourse and fixture acquisition test

## Summary

PR-089 adds a country-level acquisition test matrix for the 13-country racing inventory introduced in PR-088. The matrix checks whether the configured inventory sources provide a practical path for racecourse inventory, annual or season fixture evidence, rolling or daily updates, meeting dates, first-race times, and per-race times.

The test layer is intentionally source-path based. It confirms acquisition readiness from the PR-088 source fields without introducing live fetching, parser implementations, generated timetable records, raw source body storage, odds, payouts, predictions, or tips.

## Files

- `data/static/major-country-acquisition-test-matrix.json` — static country-by-country acquisition test results tied to PR-088 inventory source fields.
- `scripts/check-major-country-acquisition-test-matrix.mjs` — validator for matrix coverage, allowed result values, inventory source references, Singapore legacy handling, operator-split handling, and prohibited product categories.
- `docs/runbooks/pr-089.md` — this runbook.
- `package.json` — adds `validate:major-country-acquisition-test-matrix` and includes it in `npm run check` immediately after `validate:major-country-racing-inventory`.

Each country records these required acquisition-test fields:

- `update_model`
- `tested_sources_from_inventory`
- `racecourse_inventory_result`
- `annual_fixture_result`
- `rolling_update_result`
- `meeting_date_result`
- `first_race_time_result`
- `per_race_time_result`

- `next_required_collection_step`

The approved result values are:

- `source_path_exists`
- `source_path_exists_but_needs_parser`
- `operator_split_required`
- `legacy_no_active_racing`
- `source_path_missing`

## Annual vs rolling update handling

Annual or season fixture evidence and rolling or daily update evidence remain separate in the matrix:

- `annual_fixture_result` tests whether the PR-088 annual/season fixture source path can support long-range fixture acquisition.
- `rolling_update_result` tests whether the PR-088 rolling/daily/racecard source path can support near-term changes and active updates.
- `meeting_date_result`, `first_race_time_result`, and `per_race_time_result` remain separate so later acquisition work can harden date evidence independently from first-race and per-race timing evidence.

This preserves the PR-088 policy that annual fixture data is not final race-time data and that active countries need rolling/racecard evidence for final timetable timing.

## Operator-split countries

The matrix explicitly marks these countries as operator-split acquisition targets:

- United States
- Canada
- Australia
- South Africa

For these countries, the result fields use `operator_split_required` because a single country-level parser would hide materially different source systems. The next collection step is to build separate acquisition passes by operator, authority, or racing code before combining normalized country-level coverage.

## Multi-system countries

The matrix explicitly marks these countries as multi-system acquisition targets:

- Japan
- France
- New Zealand
- United States
- Canada
- Australia

Multi-system status means acquisition needs to preserve source-system boundaries during collection and normalization. Some countries are both multi-system and operator-split, such as the United States, Canada, and Australia.

## Singapore legacy handling

Singapore remains in the matrix for legacy inventory context only. It is not treated as an active timetable target.

Singapore uses `legacy_no_active_racing` for annual fixture, rolling update, meeting date, first-race time, and per-race time result fields because active horse racing ended after the final race on 2024-10-05. Its racecourse inventory result remains `source_path_exists` so Kranji and Singapore Turf Club legacy context can stay represented without implying active racing acquisition.

## What this PR does

- Adds a source-path acquisition test-result layer for all 13 major countries.
- Verifies each country against PR-088 inventory source fields instead of duplicating only racecourse inventory data.
- Separates annual/season fixture readiness from rolling/daily/racecard update readiness.
- Marks operator-split and multi-system countries explicitly.
- Preserves Singapore as a legacy, non-active timetable country.
- Wires the acquisition test validator into the existing package check pipeline.

## What this PR does not do

- Does not add live fetch runtime.
- Does not implement parsers.
- Does not generate timetable records.
- Does not store raw source response bodies.
- Does not add odds.
- Does not add payouts.
- Does not add predictions.
- Does not add tips.
- Does not add dependencies.
