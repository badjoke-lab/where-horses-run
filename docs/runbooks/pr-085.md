# PR-085 — Japan active-window candidate gap report

## Summary

PR-085 adds a Japan active-window candidate gap report that compares existing Japan JRA, NAR, and Banei candidate/generated records with the official source evidence documented in PR-084. The report records missing racecourse/date coverage and first-race-time verification needs before any further Japan timetable records are added or promoted.

## Files

- `docs/runbooks/japan-active-window-candidate-gap-report.md`
- `scripts/check-japan-active-window-candidate-gap-report.mjs`
- `docs/runbooks/pr-085.md`
- `package.json`

## Active-window basis

The report uses the existing repo candidate window where present:

- `start_date`: `2026-05-29`
- `end_date_exclusive`: `2026-06-28`
- `timezone`: `Asia/Tokyo`

PR-085 does not fetch live source pages and does not add candidate records.

## What the gap report found

- JRA has 4 candidate records only: Tokyo and Kyoto on 2026-05-30 and 2026-05-31. The other JRA inventory racecourses and other active-window dates are not verified in repo, and first-race-time values require record-by-record official verification before promotion.
- NAR has 12 candidate/generated meeting-date-level records. Exact first race times are not stored, the date-specific official workflow is identified, and active-window completeness is not checked. A future date/racecourse matrix is required before more NAR records are added.
- Banei has 3 Obihiro records for 2026-05-30, 2026-05-31, and 2026-06-01. Obihiro is the only Banei venue, exact first-race-time values exist, and those times must be rechecked before any future promotion. Active-window dates beyond the 3 current records are not verified unless official source evidence is added.

## What this does not do

- Does not add new country data.
- Does not add new candidate records.
- Does not modify generated timetable records.
- Does not promote records.
- Does not add live fetch runtime, a source page parser, or raw source body storage.
- Does not add racecards, odds, results, payouts, predictions, or tips.
- Does not add a public overlay replacement.
- Does not make complete coverage claims.
- Does not add dependencies.
