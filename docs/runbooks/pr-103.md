# PR-103 Major-country annual fixtures v0

PR-103 adds the static/manual annual and season fixture layer for the `major-countries-timetable-v0` scope that was defined by PR-102.

The data file for this PR is:

- `data/static/major-country-annual-fixtures-v0.json`

The source-of-truth planning file remains:

- `data/static/major-country-timetable-ingestion-v0.json`

## 1. What PR-103 adds

- Annual/season fixture records for the 13-country timetable v0 coverage window: `2026-05-30` through `2026-06-30`.
- Candidate meeting records keyed by country, acquisition group, racecourse, and meeting date.
- Official-source static/manual records where the annual, season, fixture-calendar, or fixture-list source exposes meeting date and racecourse information clearly enough for this PR.
- Explicit coverage-gap records for active acquisition groups where the annual source is unavailable, unstructured, or still requires manual review.
- A Singapore legacy coverage-gap record with `gap_type: no_active_racing` and `status: legacy_no_active_racing`.
- A validator, `scripts/check-major-country-annual-fixtures-v0.mjs`, wired into `npm run check` through `validate:major-country-annual-fixtures-v0`.

Every annual fixture record is a candidate meeting only. It is not final race-time data.

## 2. What PR-103 does not add

PR-103 intentionally does not add:

- Rolling/racecard final times.
- All-race times.
- Live fetch behavior.
- A parser.
- A cron job.
- Wagering, payout, prediction, or advice content.

The annual fixture layer stays static/manual and public-safe. Final race times remain pending until PR-104 or later rolling/racecard confirmation.

## 3. User-view outcome after PR-103

After PR-103, users and maintainers can know:

- Which 13 countries are in scope for the major-country timetable v0.
- Which systems/operators/acquisition groups are in scope.
- Which racecourses have scheduled candidate meetings in the v0 window where official annual/season source data is available and structured enough for this static/manual pass.
- Which groups still need manual review or have unstructured annual sources.
- That final race times are still pending until PR-104.

This should prevent a preview-style misunderstanding: the presence of annual fixture records means there is a candidate meeting date at a racecourse, not a confirmed final timetable with race times.

## 4. Operations outcome after PR-103

After PR-103, maintainers have:

- A separate annual fixture data layer apart from the PR-102 ingestion/source-plan layer.
- Candidate meeting records keyed by country/group/racecourse/date.
- Explicit coverage gaps instead of silent omissions.
- A clear input for PR-104 rolling/racecard matching.
- A guardrail against treating one record per group as completion.
- A validator that checks scope representation, required fields, Singapore legacy handling, candidate-meeting status, pending final-times status, and unsupported claim/content wording in the annual-fixture data.

PR-104 must add:

## Validation

Run:

```bash
npm run validate:major-country-annual-fixtures-v0
npm run check
```

The annual-fixture validator verifies that all countries and active groups remain represented across records and coverage gaps, Singapore remains represented as legacy/no active racing, annual records remain candidate meetings, final times remain `pending_racecard`, and no active group is silently dropped.
