# PR-090: Major-country acquisition source hardening

## Summary

PR-090 hardens the PR-089 country-level acquisition test matrix by converting broad national source paths into concrete source groups and acquisition units for each major racing country. The new source-group file defines the operator, authority, or racing-code boundary that a later sample acquisition test must use before any parser implementation is attempted.

This PR remains a static planning and validation PR. It prepares the repo for PR-091 sample acquisition fixtures without adding live fetch runtime, parsers, generated timetable records, raw source body storage, odds, payouts, predictions, tips, or dependencies.

## Files

- `data/static/major-country-acquisition-source-groups.json` — concrete source groups for all 13 major countries, including group-level source roles, parser targets, and sample acquisition units.
- `scripts/check-major-country-acquisition-source-groups.mjs` — validator for group coverage, required group IDs, field shape, annual/rolling source separation, timing-source readiness, operator splits, multi-system groups, Singapore legacy handling, and prohibited product categories.
- `docs/runbooks/pr-090.md` — this runbook.
- `package.json` — adds `validate:major-country-acquisition-source-groups` and runs it in `npm run check` immediately after `validate:major-country-acquisition-test-matrix`.

## Why this PR exists

PR-088 established a 13-country major racing inventory. PR-089 tested whether each country had source paths for racecourse inventory, annual fixtures, rolling updates, meeting dates, first-race times, and per-race times.

That was still too broad for acquisition tests. Countries such as the United States, Canada, Australia, and South Africa cannot be safely tested through one country-level parser because their acquisition paths split by operator or racing code. Other countries, such as Japan, France, and New Zealand, have multiple systems that must stay separate during collection and normalization.

PR-090 turns those country-level rows into concrete acquisition groups so PR-091 can test one group-level sample at a time.

## Operator split handling

The validator requires more than one acquisition group for the operator-split countries identified in PR-089:

- United States
- Canada
- Australia
- South Africa

These countries now use separate acquisition units instead of one country-level collection unit. Examples include Equibase versus USTA versus AQHA in the United States, Standardbred Canada versus Woodbine in Canada, Racing Australia versus Harness Australia in Australia, and NHRA versus 4Racing versus Gold Circle in South Africa.

## Annual vs rolling update handling

Annual or season fixture sources and rolling/racecard sources remain separate fields at group level:

- `annual_fixture_sources` is for long-range fixture or season-calendar evidence.
- `rolling_fixture_sources` is for near-term updates, racecards, entries, declarations, or daily race information.
- `meeting_sources`, `first_race_time_sources`, and `per_race_time_sources` remain separate so PR-091 can test each timing role explicitly.

The validator rejects merged `fixture_sources` or `timing_sources` fields and requires distinct source IDs for annual and rolling entries even when the same URL is reused with different roles.

## Singapore legacy handling

Singapore remains in scope only as legacy inventory context:

- `update_model` is `legacy_no_active_racing`.
- The only acquisition group is `singapore-turf-club-legacy`.
- Active source arrays stay empty for annual fixtures, rolling fixtures, meetings, first-race times, and per-race times.
- `parser_target` is `legacy_inventory_page`.
- `next_sample_test_target` is legacy inventory only.

No active timetable sample acquisition should be scheduled for Singapore in PR-091.

## What this PR does

- Adds concrete acquisition groups for all 13 major countries.
- Splits operator and multi-system countries into group-level acquisition units.
- Defines group-level source roles for annual fixtures, rolling fixtures, meeting dates, first-race times, and per-race times.
- Defines parser targets without implementing parsers.
- Preserves annual fixture sources separately from rolling/racecard sources.
- Preserves Singapore as a legacy, non-active racing country.
- Adds validation and wires it into the package check pipeline.

## What this PR does not do

- Does not add live fetch runtime.
- Does not implement parsers.
- Does not generate timetable records.
- Does not store raw source response bodies.
- Does not add odds.
- Does not add payouts.
- Does not add predictions.
- Does not add tips.
- Does not add dependencies.
