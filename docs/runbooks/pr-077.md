# PR-077 — UAE active-window approved candidate bundle

## Summary

This PR adds a deterministic reviewed UAE active-window candidate bundle generator.

The generator reads the existing UAE ERA candidate file and writes an approved, reviewable active-window bundle while preserving the PR-076 season-gap state when no safe UAE ERA meetings exist in the current 30-day window.

## Files

- `scripts/generate-uae-active-window-approved-candidates.mjs`
- `scripts/check-uae-active-window-approved-candidates.mjs`
- `data/candidates/uae-active-window-approved-candidates.json`
- `docs/runbooks/pr-077.md`
- `package.json`

## User-facing outcome

No direct UI change.

This PR prepares a reviewed UAE active-window candidate bundle for the next coverage-review gate. It does not replace the public timetable overlay and does not make UAE appear covered when the bundle has no records.

This PR does not publish candidate records as public coverage.

The bundle is generated from the existing safe UAE ERA candidate file:

- `data/candidates/uae-era-candidates.json`

Generated records, when present, are marked:

```text
status: source-reviewed
review_status: approved
```

## Season gap handling

The current UAE ERA candidate file can have `records: []` because no safe active-window UAE ERA meetings are present in the current 30-day window.

When the source candidate file has no records, the approved bundle preserves `records: []`, keeps explicit season gap / no active-window language, and keeps `review.promotion_target` as `null`. This reviewed empty bundle is a season-gap artifact only; it is not public coverage and must not be treated as proof that UAE has current public timetable coverage.

If safe active-window records are added in a later PR, this generator will convert them to approved reviewed candidates and point the review target at `data/generated/timetables.json` for the subsequent gate.
