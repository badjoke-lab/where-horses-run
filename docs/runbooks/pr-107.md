# PR-107 — All race timetable capture reality gate

PR-107 restores the real target: a confirmed timetable record must include meeting date, racecourse, first race time, and every official race time for that meeting.

Fixture dates, racecourses, annual calendars, one-race records, source links, pending records, and coverage gaps are not enough.

## Required rule

A record may only be treated as a confirmed timetable record when it has:

- country_id
- group_id
- racecourse
- meeting_date
- first_race_time
- races[] with all official races for that meeting
- official source_url
- source_type from racecard, entries, declarations, daily info, fields, or acceptances
- source_capture_date
- last_checked

If races[] has only one race while the meeting has more races, it is not all-race captured.

## User result

Users must be able to distinguish full all-race timetable records from fixture-only or pending records.

## Operations result

Maintainers must prioritize source-specific extraction work by missing all-race data, not by docs, gates, or pending record counts.
