# PR-288 — Align country, source, and racecourse identifiers for entries 01–12

## Summary

This PR freezes the canonical identifier foundation for the first twelve reviewed country-page deliveries before production profile-v2 records are added.

The machine-readable inventory is:

```text
data/static/country-page-id-inventory-01-12.json
```

It records:

- the tracker-linked country ID and slug for deliveries 01–12,
- canonical organiser and distributor source IDs,
- canonical racecourse IDs,
- the proposed principal racecourse set,
- whether each ID is already registered, reserved for a later profile batch, or still only a candidate,
- explicit pending states where the reviewed evidence does not support a final claim.

## Evidence boundary

The inventory is derived from the twelve reviewed country notes already in the repository. This PR does not perform a new external source test and does not claim that an observed racecourse list is a complete current national inventory.

Bahrain and Qatar retain an explicit pending principal-racecourse state because their reviewed source-test summaries did not establish a reliable venue name. Morocco and Mexico also retain pending principal-racecourse selection until publication-time research confirms the appropriate current scope. Oman retains a pending organiser state because the reviewed evidence did not settle which body currently schedules and stages the national flat-racing programme.

## Registered and reserved IDs

`registered` means the ID already exists in the relevant public static registry and its country reference must match.

`reserved` means the canonical ID is fixed for a later profile batch but the full public registry record has not yet been materialized.

`candidate` means the reviewed evidence identified the source or venue as a candidate, but it must not be treated as a confirmed current public record.

The next three profile PRs will materialize only the source and racecourse records that have enough reviewed content for public use.

## Validation

Run:

```text
node scripts/check-country-page-id-inventory-01-12.mjs
```

The validator checks:

- delivery order is exactly 01–12,
- country IDs and slugs are unique and identical,
- tracker rows, note references, page kinds, and statuses still match,
- this infrastructure PR does not advance profile status,
- registered IDs resolve to the existing static registries,
- reserved and candidate IDs do not collide with existing records,
- source and racecourse IDs are unique,
- every country/source/racecourse reference resolves inside the inventory,
- organiser and distributor roles remain separate,
- confirmed organiser and principal-racecourse states have IDs,
- pending states do not silently claim IDs,
- principal racecourse IDs are a subset of the country racecourse inventory.

The dedicated workflow also runs the full Astro build.
