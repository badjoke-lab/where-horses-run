# PR-067 — Japan JRA candidate generator

## Summary

This PR adds the first Japan candidate generator for JRA.

The generator is intentionally dry-run only. It reads the existing safe JRA meeting-level timetable seed and writes a reviewable candidate file.

## Files

- `scripts/generate-japan-jra-candidates.mjs`
- `data/candidates/japan-jra-candidates.json`
- `scripts/check-japan-jra-candidate-generator.mjs`
- `package.json`
- `docs/runbooks/pr-067.md`

## User-facing outcome

No direct UI change.

This PR moves JRA data handling away from pure hand-edited seed data and toward candidate generation.

## Safety boundary

This PR does not add live fetch.
This PR does not add source page parsing.
This PR does not store source response bodies.
This PR does not publish candidate records as coverage.

The candidate file remains:

```text
review_status: needs_review
status: candidate
extraction_method: adapter_dry_run
```

## Validation

Run:

```sh
npm run generate:japan-jra-candidates
npm run validate:japan-jra-candidate-generator
npm run build
```

The validator also checks that the generated candidate output is not stale.
