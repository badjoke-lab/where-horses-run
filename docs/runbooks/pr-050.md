# PR 050

Status: Timetable schema v0 and Japan generated fallback.

## Scope

- Start the real timetable data foundation for Today, Tomorrow, and 30-day views.
- Add `data/generated/timetables.json` as the safe generated timetable file.
- Add timetable schema v0 documentation.
- Add a Japan generated coverage fallback runbook.
- Add a validator for safe timetable schema boundaries.

## Safety boundary

- This PR is not a racecard parser.
- This PR does not enable live fetch.
- This PR does not store official page bodies, raw HTML, response bodies, or copied official page text.
- This PR does not store entries, horse names, jockey names, odds, results, payouts, prediction content, or tips.
- Official source links remain the final confirmation point.

## User-facing outcome

Users may not yet see the full Today / Tomorrow UI after this PR.

The project now has the safe data structure required to show:

- Country.
- Racecourse.
- Date.
- Start time.
- Official source.
- Last checked time.
- Status.
- Confidence.

## Validation

- `npm run validate:timetable-schema-v0` checks the new generated timetable file and rejects unsafe fields or raw markup markers.
- `npm run check` includes the new timetable schema validator.
