# PR-051 — Display safe timetable data on country pages

## Summary

This PR makes the generated safe timetable foundation visible on country pages in English and Japanese.
It reads `data/generated/timetables.json` through `siteData.generated.timetables` and displays the current timetable state for the country being viewed.

## User-facing behavior

Country pages now show a **Safe timetable data** card that distinguishes between:

- matching verified timetable records,
- matching generated fallback data, and
- no safe timetable records for that country yet.

The UI includes the country ID, mode, verified record count, timetable source links where available, fallback/manual state, generated time, and notes.

## Japan current state

Japan currently has a manual fallback structure only:

- mode: `manual-fallback`
- verified records: `0`
- official source: `japan-jra-home`
- final confirmation: use the official JRA link

This means Japan has safe timetable structure available, but no verified meeting dates or start times are listed yet.

## Safety boundary

Live fetching remains disabled. This PR does not add a live fetcher, parser, source page fetching, raw HTML storage, response body storage, or racecard extraction.

Country timetable UI must not display racecard, entry, horse, jockey, odds, result, payout, prediction, or tip content as timetable data.
