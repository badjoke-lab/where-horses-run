# PR-101 — Combine major-country preview timetable batches

## Summary

PR-101 makes all 24 active static manual preview samples visible on the major-country timetable preview page. It combines the PR-098 batch 001 samples and the PR-100 batch 002 samples into one dependency-free server-rendered preview data flow.

PR-101 does not provide live coverage. PR-101 does not provide full country/date/racecourse coverage. Singapore remains legacy/no active racing.

## Files

- `src/data/majorCountryPreviewTimetableSamples.ts` combines batch 001 and batch 002 preview records and exports the summary used by the page.
- `src/pages/major-countries/preview-timetable.astro` renders the combined summary, status wording, and preview sample component.
- `src/components/PreviewTimetableSamples.astro` groups the static sample cards by country for a 24-card view.
- `scripts/check-combined-preview-timetable-ui.mjs` validates the combined UI behavior and guardrails.
- `package.json` adds `validate:combined-preview-timetable-ui` to `npm run check` immediately after `validate:preview-timetable-samples-batch-002`.
- `docs/runbooks/pr-101.md` records the operational notes for this PR.

## Current schedule and current position

The major-countries work is still pre-v0. The current position is a static manual preview page that exposes known sample meeting dates and race times from official/approved sources so users can see the display shape for future timetable work. These preview samples are not production timetable coverage and are not sufficient for the 13-country timetable MVP.

This is not the v0 release gate. PR-102 should add the major-country timetable ingestion v0 source plan, lock the 13-country/24-active-group/Singapore-legacy scope, and make the path to production timetable data explicit.

## What PR-100 completed

PR-100 added batch 002 preview samples for the remaining 18 active acquisition groups. Those records completed the static manual sample set but were not yet visible together with the PR-098 batch 001 records on the user-facing preview page.

## What PR-101 adds

PR-101 adds combined batch handling for the preview timetable UI:

- Batch 001 records are loaded from `data/static/preview-timetable-samples-batch-001.json`.
- Batch 002 records are loaded from `data/static/preview-timetable-samples-batch-002.json`.
- The combined export contains exactly 24 active group records.
- The combined records are sorted by country, system, meeting date, and racecourse for stable display.
- The preview summary reports total samples, countries, systems, source capture dates, active group count, legacy group count, and static-sample notices.

## User-visible behavior

Users visiting `/major-countries/preview-timetable/` can now see all 24 active static manual preview samples. The page displays:

- Total samples: 24.
- Active group count: 24.
- Countries represented.
- Systems represented.
- Source capture dates represented.
- Static manual sample notices.
- Official source links on each preview card.
- Race lists for each sample.
- A plain-language status section explaining what is available now, what is not available yet, and what comes next.

## Operational behavior

The page remains static and dependency-free. It does not add client-side fetch behavior, parser implementation, scraping dependencies, generated live timetable records, or raw source body/html storage.

The preview data is loaded at build/render time through the combined data module. The UI remains server-rendered Astro markup.

## Combined batch handling

The combined module exports:

- `allPreviewTimetableSamples` — all records from batch 001 and batch 002, sorted for stable display.
- `previewTimetableSummary` — aggregate counts and user-facing notices for the static manual preview set.

The combined set includes the 6 batch 001 active groups and the 18 batch 002 active groups.

## Singapore legacy handling

Singapore remains legacy/no active racing. It is not included as active timetable coverage and is not counted as an active group in the combined preview set.

## What users can do now

Users can:

- View static sample meeting dates and race times from official/approved sources.
- Open official source links for each preview sample.
- Compare the sample shape across all 24 active acquisition groups.

## What users still cannot do

Users still cannot:

- Browse complete live timetables.
- See every date/racecourse for every country.
- Rely on automatic updates.
- Treat this preview as full country/date/racecourse coverage.

## What this PR does not do

PR-101 does not add:

- Live fetch runtime.
- Parser implementation.
- Scraping dependency.
- Raw source body/html storage.
- Generated live timetable records.
- Odds, payouts, predictions, or tips.
- New dependencies.
- Full country/date/racecourse coverage.
