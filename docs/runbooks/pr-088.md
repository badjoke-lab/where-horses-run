# PR-088: Major-country racing inventory

## Summary

PR-088 adds static racing-inventory source data for the 13 countries in `major-countries-v0`. The inventory is source-oriented: it records authorities, operators, racecourse inventory coverage, annual fixture sources, rolling fixture sources, meeting sources, and final race-time source categories without fetching live data or generating timetable records.

## Files

- `data/static/major-country-racing-inventory.json` — required JSON seed for the major-country racing inventory.
- `scripts/check-major-country-racing-inventory.mjs` — validator for schema, country coverage, non-placeholder content, legacy Singapore handling, and prohibited product categories.
- `docs/runbooks/pr-088.md` — this runbook.
- `package.json` — adds `validate:major-country-racing-inventory` and includes it in `npm run check` after `validate:major-countries-v0-scope`.

## Annual fixture vs rolling update model

Annual fixture sources and rolling update sources are intentionally separate fields.

- `annual_fixture_sources` identifies season, annual, or long-range calendar sources. These are useful for racecourse/meeting planning but are not treated as final race-time truth.
- `rolling_fixture_sources` identifies active, near-term, daily, weekly, racecard, declarations, nominations, or operator-update sources that can refine meeting status and timing.
- `meeting_sources`, `first_race_time_sources`, and `per_race_time_sources` remain separate so later acquisition work can distinguish meeting existence, first-race time, and per-race timing evidence.

This separation preserves the policy that annual fixture data is not final race-time data and that racecard or daily information is required for final race times.

## Singapore legacy handling

Singapore is the only country marked with `legacy_no_active_racing`. Its racecourse inventory remains present for Kranji and Singapore Turf Club legacy context, but its active racing fixture/time source arrays are empty because horse racing ended after the final race on 2024-10-05.

The validator enforces that Singapore is the only legacy/no-active-racing country and that the active fixture/time source arrays remain empty for Singapore.

## What this PR does

- Adds real static inventory data for all 13 major countries.
- Preserves separate annual fixture, rolling fixture, meeting, first-race-time, and per-race-time source fields.
- Adds a validator that confirms JSON parsing, required countries, required fields, non-empty active-country source coverage, Singapore legacy handling, placeholder rejection, and prohibited products.
- Wires the validator into the package check pipeline immediately after the major-countries v0 scope validator.

## What this PR does not do

- Does not add live fetches.
- Does not add parser implementations.
- Does not generate timetable records.
- Does not add odds.
- Does not add payouts.
- Does not add predictions.
- Does not add tips.
