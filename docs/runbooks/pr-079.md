# PR-079: M3 v0 release gate

## Summary

PR-079 adds an M3 v0 release gate for the internal generator-based timetable foundation milestone.

The gate verifies that Japan, Hong Kong, and UAE candidate-generation work is wired into package validation, that required candidate and approved-bundle files exist, and that the public status wording remains clear about partial public coverage.

## Files

- `scripts/check-m3-v0-release-gate.mjs` — release-gate validator for package scripts, candidate/bundle safety, public wording, country coverage claims, and UAE season-gap handling.
- `data/candidates/japan-active-window-approved-candidates.json` — generated Japan approved active-window candidate bundle required by the release gate.
- `package.json` — adds `validate:m3-v0-release-gate` and runs it in `npm run check` after `validate:cross-country-candidate-validator`.
- `docs/runbooks/current-status.md` — updates public-safe status after PR-078 for the M3 v0 generator foundation release gate.
- `docs/runbooks/pr-079.md` — this runbook.

## User-facing outcome

No broad UI change is introduced.

The user-facing outcome is clearer status documentation: the project can describe the M3 v0 generator foundation as internally release-gated while still saying that public coverage remains partial.

## Release gate checks

The release gate checks that package validation scripts exist for:

- `validate:japan-jra-candidate-generator`
- `validate:japan-nar-candidate-generator`
- `validate:japan-banei-candidate-generator`
- `validate:hong-kong-active-window-approved-candidates`
- `validate:uae-era-candidate-generator`
- `validate:uae-active-window-approved-candidates`
- `validate:cross-country-candidate-validator`

It also checks that `npm run check` includes `validate:cross-country-candidate-validator` and then runs `validate:m3-v0-release-gate` after it.

Required candidate and bundle files:

- `data/candidates/japan-active-window-approved-candidates.json`
- `data/candidates/hong-kong-active-window-approved-candidates.json`
- `data/candidates/uae-active-window-approved-candidates.json`
- `data/candidates/hong-kong-hkjc-candidates.json`
- `data/candidates/uae-era-candidates.json`

Candidate and bundle checks include:

- `schema_version: timetable-candidates-v0`
- top-level `candidate_window`
- top-level review metadata
- no public complete coverage claim
- no forbidden markers for raw source bodies, racecards, entries, names, odds, results, payouts, predictions, or tips

Public wording checks include:

- current phase is the M3 v0 generator foundation release gate
- M3 v0 generator foundation is complete
- public coverage remains partial
- UAE is season gap / no active-window meetings
- no live fetch is enabled
- no source page parsing
- no raw source body storage
- no public complete coverage claim
- no racecards, odds, results, payouts, predictions, or tips are stored

## Safety boundary

This PR does not add:

- live fetch
- source page parsing
- raw source body storage
- racecards
- odds
- results
- payouts
- predictions
- tips
- public overlay replacement
- broad UI changes
- new dependencies

## Season gap handling

The UAE active-window approved bundle may keep `records: []` only when the file clearly documents the season gap, states that there are no active-window meetings, and states that the empty bundle is not public coverage.

This preserves the PR-077 state: the release gate can pass during the UAE season gap without treating an empty approved bundle as active public coverage.

## Validation commands

Run these commands before merging:

```bash
node scripts/check-m3-v0-release-gate.mjs
npm run build
npm run check
```
