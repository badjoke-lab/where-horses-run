# PR-105 — Major country timetable v0 UI

PR-105 merges the PR-103 annual fixture candidate layer and the PR-104 rolling/racecard confirmation layer into a user-facing static/manual timetable v0 data file and page.

## 1. What PR-105 adds

- Merged user-facing timetable data in `data/static/major-country-timetable-v0.json`.
- Static/manual timetable v0 page at `/major-countries/timetable/`.
- Confirmed race times where official rolling/racecard data exists.
- Pending statuses where official racecard/time data is not yet published.
- Source trace from the annual and rolling layers so maintainers can distinguish candidate meetings from confirmed records.

## 2. What PR-105 does not add

- No live fetch.
- No parser.
- No cron.
- No full automation.
- No live coverage.
- No odds.
- No payouts.
- No predictions.
- No tips.
- No full racecard republication.

## 3. User-view outcome after PR-105

After PR-105, users can:

- Open `/major-countries/timetable/`.
- Choose or browse by country.
- Choose or browse by system/operator.
- Browse by date.
- Check racecourse.
- See first-race time where officially confirmed.
- See all race times where officially confirmed.
- See pending status where racecard/time data is not yet published.
- See official annual source links.
- See official rolling/racecard source links where available.
- See `source_capture_date`, `last_checked`, and stale/pending status.

## 4. Operations outcome after PR-105

After PR-105, maintainers have:

- Annual fixture candidate layer.
- Rolling/racecard confirmation layer.
- Merged timetable display layer.
- A user-facing route backed by static/manual data.
- Explicit source trace.
- Explicit pending/gap states.
- A ready target for the PR-106 completion gate.
