# PR-287 — Make country detail pages consume profile v2

## Summary

This PR introduces the production runtime boundary for country profile v2.

Country detail pages continue to use the shared `CountryDetailPage` component,
but the profile getter now reads a normalized runtime collection built in this
order:

1. canonical profile-v2 records,
2. narrowly allowlisted legacy compatibility records.

## Canonical input

New profile-v2 records belong in:

```text
data/static/country-profiles-v2.json
```

Each record must satisfy `schemas/country-profile-v2.schema.json` and the
existing tracker-linked validator before it can be used by a page.

The collection begins empty in this infrastructure PR. The following PRs add
reviewed profiles in delivery batches.

## Compatibility boundary

Only these existing reviewed seeds may use the legacy profile format:

```text
japan
hong-kong
```

The allowlist is defined in `src/lib/country-profile-runtime.ts`. No other
country can silently fall back to `data/static/country-profiles.json`.

When a v2 record exists for an allowlisted country, the v2 record takes
precedence automatically and the legacy record is not included in the runtime
collection.

## Page behavior

The adapter converts profile v2 into the stable page-facing model currently
used by `CountryDetailPage`:

- bilingual hero and overview copy,
- racing types and systems,
- seasonality and schedule guidance,
- surfaces and track formats,
- official source references,
- coverage notes and review metadata.

Organiser and distributor source IDs remain separate in the normalized record.
Their union is used only for resolving the existing official-source cards.

The change does not expand timetable output. Country pages remain
one-meeting-per-row views, and per-race programme details remain outside country
pages.

## Validation

The dedicated runtime check verifies:

- the v2 collection is an array with unique identities,
- every added v2 record passes the profile-v2 validator,
- the legacy allowlist is exactly Japan and Hong Kong,
- v2 records override legacy records,
- organiser and distributor roles are retained,
- both language routes render the shared country detail component,
- prohibited participant, wagering, result, payout, raw-source and direct-stream
  fields are absent from the v2 collection,
- the complete Astro site still builds.

## Programme impact

This is an infrastructure PR. It does not migrate production country profiles
or advance tracker rows.

```text
published: 0
note_reviewed: 12
profile_ready: 2 legacy seeds
not_started: 84
```
