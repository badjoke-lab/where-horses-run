# PR-054 — 30-day calendar using safe timetable data

## Summary

The 30-day calendar pages now use the generated safe timetable dataset from `data/generated/timetables.json` through `siteData.generated.timetables`.

The English `/calendar/` page and Japanese `/ja/calendar/` page compute a 30-day range from the timetable `generated_at` date. Records are included when their safe `date` value is on or after the generated date and before generated date plus 30 days.

## User-facing outcome

- Safe timetable records are grouped by date when matching records exist in the 30-day range.
- Each displayed record is limited to safe timetable fields such as country, racecourse, date, start time, timezone, racing type, official source, last checked, status, confidence, and notes.
- When no matching records exist, the page shows a clear empty state.
- Official source links remain the final confirmation point for users.
- Live fetching remains disabled.

## Safety boundary

This PR is UI-only.

- No parser is added.
- No live source fetch is added.
- No source page fetching is added.
- No raw source body storage is added.
- No racecard, entries, odds, results, payout, prediction, or tip content is displayed.

## Validation

Run these checks with Node v22.22.2 or compatible:

```sh
npm run validate:calendar-30d-timetable-ui
npm run validate:timetable-schema-v0
npm run check
```
