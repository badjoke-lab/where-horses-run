# PR-070 — Candidate to overlay promotion script

## Summary

This PR adds a generic promotion script that converts reviewed timetable candidate files into overlay-style output.

It does not promote the current JRA, NAR, or Banei candidates. Those files remain `needs_review`.

## Files

- `scripts/promote-timetable-candidates.mjs`
- `scripts/check-timetable-candidate-promotion.mjs`
- `data/candidates/promotion-approved.sample.json`
- `data/candidates/promotion-approved.overlay.sample.json`
- `package.json`
- `docs/runbooks/pr-070.md`

## User-facing outcome

No direct UI change.

This PR adds the safe bridge between candidate generation and production-facing overlay data.

## Safety boundary

The promotion script only promotes records with:

```text
review_status: approved
```

It excludes:

```text
review_status: needs_review
review_status: rejected
```

It also rejects records containing unsafe markers such as:

- raw HTML
- source body
- racecard
- entries
- horse names
- jockey names
- trainer names
- odds
- results
- payouts
- predictions
- tips
- betting advice

## Validation

Run:

```sh
npm run validate:timetable-candidate-promotion
npm run build
```

Manual example:

```sh
npm run promote:timetable-candidates -- --input data/candidates/promotion-approved.sample.json --output data/candidates/promotion-approved.overlay.sample.json --check
```
