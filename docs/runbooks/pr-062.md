# PR-062 — Japan active timetable records

## Summary

This PR adds active-window Japan meeting-level timetable records beyond the existing JRA seed.

It adds a generated overlay file:

- `data/generated/japan-active-timetable-records.json`

The overlay is merged into `siteData.generated.timetables` through `src/lib/data.ts` so existing country and calendar views can use the combined record list.

## User-facing outcome

Japan now has active-window meeting-level records for:

- NAR local meetings
- Banei meetings

The previous JRA records remain in the base timetable file.

## Important limitation

This PR still does not claim full Japan coverage.

It does not add complete JRA coverage.
It does not add complete NAR coverage.
It does not add complete Banei coverage.
It adds active-window meeting-level records only.

## Safety boundary

Only safe meeting-level fields are added:

- country
- racecourse
- date
- start-time text when safely stored
- timezone
- racing type
- source id
- source URL
- status
- confidence
- notes

No source body is stored.
No live fetch is added.
No parser is added.
No detailed race data is added.

## Validation

Validator added:

```sh
node scripts/check-japan-active-timetable-records.mjs
```

`package.json` wiring may need a follow-up if the contents API blocks the package update.
