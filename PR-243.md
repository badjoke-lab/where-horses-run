# PR-243: Publication display policy resolver

## Summary

This PR performs roadmap PR-5: separate canonical capability rank from effective public rank.

## Added

- `src/data/publicationDisplayPolicies.ts`
- `src/lib/timetable/publicationPolicy.ts`
- `scripts/check-publication-policy-resolver.mjs`

## Initial publication policy

A+ capability may publish as A+ during the initial display test period.

Reviewed authority families currently allow a maximum public rank of A+:

- HKJC
- JRA
- NAR
- Banei
- Emirates Racing Authority

When effective public rank is A+, the resolver may expose these approved programme-summary fields:

- race name
- distance
- surface
- course label

Lower-capability records are never promoted. A B record remains B even when its policy maximum is A+.

Unknown or unreviewed source families use the conservative default policy and are capped at C.

## Switching behavior

The same resolver can switch A+ down to A, B+, B, C, D, or not_listed.

This allows the project to test A+ display first, then test policy downgrades without changing canonical data or acquisition output.

## Scope

No public JSON is generated.

No page input is changed.

No canonical data is modified.

No source fetching or snapshot acquisition is added.

No Live / Replay label is enabled in the initial policies.
