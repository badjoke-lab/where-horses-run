# PR-185: JRA section split schema

## Current PR

This PR applies the corrected B+ / A policy to the JRA section split design.

It separates two outputs:

- B+ summary output for calendar/day summaries.
- A detail output for meeting detail pages.

## Added

- `data/generated/timetable/pr-185-jra-section-split-schema.json`
  - Defines the section split problem.
  - Defines `meeting_summary_record` for monthly calendar and day summary.
  - Defines `meeting_detail_record` for A-capable detail pages.
  - Keeps B+ as first_race_time plus last_race_time only.
  - Keeps race rows out of monthly calendar output.

## Section split rule

One JRA calendar page can contain multiple official meeting sections.

Rows must be split by official section before any summary fields are derived.

## B+ output

B+ output is `meeting_summary_record` only.

Required B+ fields:

- first_race_time
- last_race_time

B+ does not require race rows.

## A output

A output has two parts:

- `meeting_summary_record`
- `meeting_detail_record`

The monthly calendar uses only the summary record.

The detail page uses the detail record and race rows.

## Safety

- No JRA promotion.
- No calendar rows added.
- No detail pages added.
- No rank change.
- No race rows in monthly calendar.
- No B+ race-by-race requirement.
