# PR-250: Migrate timetable list pages to public view

## Summary

This PR performs roadmap PR-7: migrate the main timetable list pages from mixed legacy and transitional inputs to the public timetable view.

## Changed

- `src/data/timetableMeetingRows.ts` is now a compatibility adapter over `publicTimetableViewModel.ts`.
- `src/components/TimetableMeetingList.astro` supports nullable detail links and English/Japanese labels.
- English calendar, today, tomorrow, and current timetable pages keep their existing layouts but now receive public-view rows.
- Japanese calendar, today, and tomorrow pages now use the same shared list component and public-view adapter.
- `astro.config.mjs` runs the canonical builder and public-view builder before Astro loads page data.
- `scripts/check-public-timetable-list-page-migration.mjs` verifies the migration boundary.

## Result

English and Japanese calendar, today, and tomorrow pages now use the same public timetable adapter.

The current timetable page also reads the public timetable adapter.

No page imports legacy timetable JSON directly.

List pages remain one meeting per row and never expand race-by-race timetable rows.

Detail links are shown only when the public view provides a detail path.

Official-source links come from the public view.

## Generation order

Astro configuration runs:

1. `scripts/timetable/build-canonical-timetable.mjs`
2. `scripts/timetable/build-public-timetable-view.mjs`
3. Astro page build

This prevents committed bootstrap JSON from becoming the permanent page input when source data changes.

## Scope

- No country page timetable section is migrated in this PR.
- No racecourse page timetable section is migrated in this PR.
- No meeting detail page is migrated in this PR.
- No legacy input file is deleted.
- No source acquisition route is added.
