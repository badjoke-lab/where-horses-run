# PR-184: Add public display boundary policy guard

## Summary

This PR adds machine-readable policy data and a validation guard for the timetable public display boundary already present in `docs/specs/timetable-public-display-boundary.md`.

## Added

- `data/generated/timetable/pr-184-public-display-boundary.json`
  - Adds a machine-readable policy record for C / B / B+ / A / A+ display boundaries.
  - Defines list-page, detail-page, A+, and Live / Replay display boundaries.
  - Records explicit non-promotion decisions.
- `scripts/check-pr-184-public-display-boundary.mjs`
  - Adds a validation guard for the policy record and existing public display boundary spec.

## Display boundary

C, B, and B+ remain meeting-summary ranks.

A can show race label or race number plus post time.

A+ can show limited programme summary fields only when routed to meeting detail pages.

A+ detail is limited to meeting detail pages.

List pages remain one meeting per row.

## Live / Replay boundary

Live / Replay policy is separate from timetable rank.

Allowed labels are limited to official-source routing and access-condition labels.

The site does not host, embed, mirror, or redistribute video.

## Safety

- No new source records.
- No rank promotion.
- No calendar rows added.
- No meeting detail UI implementation.
- No runtime fetching.
- No parser, scraper, adapter, scheduler, or writeback logic.
- No raw source storage.
- No entries, odds, results, payouts, predictions, tips, full racecards, or video embeds.
