# PR-269: HKJC full route normalizer

## Summary

This PR starts roadmap PR-16 by making the HKJC normalizer use the full June route config as the meeting set.

## Changed

- `scripts/timetable/normalize-hkjc-racecards.mjs`
- `scripts/check-hkjc-full-route-normalizer.mjs`

## Result

HKJC normalizer now uses the route config as the full June meeting set.

Snapshot-missing meetings are kept as C records instead of disappearing.

The normalized output records the route meeting count and snapshot observation count.

Extraction summaries now mark whether each meeting was present in the current source snapshot.

This prevents a partial source snapshot from silently shrinking the HKJC June meeting set.

## Scope

No country page is changed.

No racecourse page is changed.

No source fetching behavior is changed.

No canonical builder behavior is changed in this PR.

No public view output is regenerated in this PR.

## Next

Next step is canonical/public integration of the full HKJC normalized record set.
