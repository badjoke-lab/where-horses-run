# PR-191: JRA calendar and day-page display adapter

## Current PR

This PR defines the display adapter contract for normalized JRA section records.

It uses the normalized `summary_records` from PR-190 for monthly calendar and day-page summary output.

A race rows remain detail-page data only.

## Added

- `data/generated/timetable/pr-191-jra-calendar-day-display-adapter.json`
  - Defines monthly calendar adapter input and output.
  - Defines day-page adapter input and output.
  - Keeps monthly and day summary views on `summary_records` only.
  - Keeps `detail_records` for meeting detail pages only.
  - Keeps B+ as first_race_time plus last_race_time.
  - Keeps A detail access through `detail_page_path`.

## Monthly calendar display

Monthly calendar consumes `summary_records` only.

It may show:

- meeting_id
- meeting_date
- system
- racecourse
- rank
- first_race_time
- last_race_time
- race_count
- detail_page_path
- source_url
- verification_status
- last_checked

It must not include race rows.

## Day page display

Day page consumes `summary_records` only for meeting summaries.

A records may show a detail link, but race rows still belong to the meeting detail page.

## Scope

- No JRA promotion.
- No rank gate applied.
- No automatic fetch route added.
- This PR adds adapter contract only.
