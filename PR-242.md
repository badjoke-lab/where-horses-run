# PR-242: Existing timetable JSON to canonical conversion

## Summary

This PR performs roadmap PR-4: convert the existing timetable inputs into canonical meeting and meeting-detail outputs.

## Added

- `scripts/timetable/build-canonical-timetable.mjs`
- `data/generated/timetable/canonical/meetings.json`
- `data/generated/timetable/canonical/meeting-details.json`
- `scripts/check-canonical-timetable-output.mjs`

## Inputs

Meeting summary inputs:

- `data/generated/timetables.json`
- `data/generated/japan-active-timetable-records.json`
- `data/generated/normalized-timetable.json`

Meeting detail inputs:

- `src/data/normalizedTimetableMeetingDetails.ts`
- `data/generated/timetable/hkjc-normalized-meeting-details.sample.json`

## Result

The builder creates one canonical meeting dataset and one canonical meeting-detail dataset.

- Legacy/manual meeting seeds are converted into canonical meeting records.
- Manually reviewed normalized records override matching legacy meeting IDs.
- JRA public-safe post-time rows become an A canonical detail record.
- HKJC programme-summary rows become an A+ canonical detail record.
- Meeting IDs use authority-specific prefixes for JRA, HKJC, NAR, Banei, and UAE records.
- Validation runs the builder and requires at least 34 canonical meetings and exactly 2 canonical detail records.

## Scope

No public view is generated.

No publication policy resolver is added.

No page input is changed.

No source fetching or snapshot acquisition is added.

No legacy input is deleted.
