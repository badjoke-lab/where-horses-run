## Current position

The monthly/current timetable must stay at one row per meeting.

It should expose:

- Rank
- Time
- Race count
- Details
- Source

A-rank records must not expand race-by-race rows on the monthly list. Race-by-race display belongs on a later meeting detail page.

## Display rules

- C: `Time unknown` / `Not captured` / `Source`
- B: `First 10:05` / `Not captured` / `Source`
- B+: `10:05–16:30` / `Not captured` / `Source`
- A: `10:05–16:30` / `12 races` / `View meeting`

`last_race_time` means the final race start time, not a meeting end time.

## Scope

- Update the current timetable record cards to be rank-aware.
- Remove monthly race-by-race expansion from the listing component.
- Keep source trace and freshness available under details.

- No new calendar records are promoted.
